逐段双语理解 / Paragraph-Level Bilingual Coverage

本教程解决的不是复杂 composition，也不是完整建模流程，而是 USD Python 入门的最小闭环：创建一个新的 `Usd.Stage`，在 stage 中写入两个 typed prim，然后把 root layer 保存为 `HelloWorld.usda`。读者只要能理解这三个动作，就已经掌握了后续教程反复使用的 stage、layer、prim path 和 schema authoring 基础。

The tutorial walks through creating a simple USD stage containing a transform and a sphere.

第一步是环境与脚本位置。官方建议先打开 `USD/extras/usd/tutorials/helloWorld/helloWorld.py`，并确认 `Configure your Environment` 中的 Python 环境已经能导入 `pxr`。如果 `from pxr import Usd, UsdGeom` 失败，优先检查 OpenUSD 安装、`PYTHONPATH`、shell 环境和当前工作目录；不要改写教程里的 API 名称。

Open `USD/extras/usd/tutorials/helloWorld/helloWorld.py` to see the Python script that creates and exports the stage.

`from pxr import Usd, UsdGeom` 引入两个核心模块：`Usd` 提供 stage、layer 和 scene description 的基础入口，`UsdGeom` 提供几何 schema，例如 `UsdGeom.Xform` 和 `UsdGeom.Sphere`。这里的模块名必须原样保留，因为后续代码会直接调用这些 Python API。

The script imports `Usd` and `UsdGeom` from `pxr`.

`Usd.Stage.CreateNew('HelloWorld.usda')` 创建一个新的可编辑 stage，并把它的 root layer 绑定到 `HelloWorld.usda`。stage 是应用程序操作场景描述的编辑入口；layer 是最终落盘的持久化载体。初学者常把 stage 误解为文件本身，但更准确地说，stage 是组合和编辑视图，root layer 才是这一步创建出的 `.usda` 输出目标。

`Usd.Stage.CreateNew('HelloWorld.usda')` creates a new stage with a root layer that can be saved as an ASCII USD file.

`UsdGeom.Xform.Define(stage, '/hello')` 在 prim path `/hello` 上定义一个 `Xform` typed prim。`Xform` 是用来承载变换和组织层级的 prim，教程中它作为父节点，为后面的 sphere 提供 namespace 层级。这里的 `/hello` 不是文件路径，而是 USD scene namespace 中的 prim path。

`UsdGeom.Xform.Define(stage, '/hello')` defines a transform prim at the `/hello` prim path.

`UsdGeom.Sphere.Define(stage, '/hello/world')` 在 `/hello/world` 上定义一个 `Sphere` typed prim。这个路径表示 `world` 是 `/hello` 的子 prim；也就是说，教程用两行 `Define` 同时展示了 typed schema authoring 和父子路径关系。`Define` 的职责是创建或确保 typed prim 存在，而不是只查询已经存在的 prim；查询路径通常会用 `GetPrimAtPath` 之类 API。

`UsdGeom.Sphere.Define(stage, '/hello/world')` defines a sphere prim under `/hello`.

最后一步 `stage.GetRootLayer().Save()` 把内存中的 root layer 写入磁盘。如果省略这一步，代码虽然创建了 stage 和 prim，但不会产生可检查的 `HelloWorld.usda` 文件。教程的闭环判断很直接：运行脚本后，当前目录应该出现 ASCII USD 文本层，打开后能看到 `/hello` 和 `/hello/world` 的定义。

`stage.GetRootLayer().Save()` saves the root layer so the authored scene description is written to `HelloWorld.usda`.

本页的主要 section 是 `Python for USD`。它不是要求读者学习全部 Python API，而是让读者看到 USD authoring 的最小顺序：import 模块、创建 stage、定义父 prim、定义子 prim、保存 layer。这个顺序不要调整，因为后续引用、遍历、属性创作和变体教程都会以类似 stage 为基础继续扩展。

The `Python for USD` section establishes the minimal order for authoring with Python: import modules, create a stage, define prims, and save the layer.

调试时应分层定位：如果 import 失败，是环境配置问题；如果 `HelloWorld.usda` 没生成，检查工作目录和 `Save()`；如果文件存在但看不到 sphere，检查 prim path 字面量是否保持 `/hello/world`；如果在 `usdview` 中显示异常，再检查 OpenUSD 版本、schema 插件和可视化工具，而不是先怀疑教程的 scene description 语义。

Debugging should separate environment, save location, path literals, and visualization issues.

与相邻教程的关系也很重要：`Inspecting and Authoring Properties` 会继续说明如何查看和写属性；`Referencing Layers` 会把已有 layer 用作引用；`Traversing a Stage` 会遍历 stage 中的 prim。`Hello World` 的作用就是给这些后续主题提供一个足够小、可重复生成、容易打开检查的 USD 文件。

Later tutorials build on this simple stage to inspect properties, reference layers, and traverse scene contents.

运行完成后的验证方式也很具体：`HelloWorld.usda` 应该是 ASCII USD layer，可以用文本编辑器打开；其中应能看到一个 `/hello` prim 和一个 `/hello/world` prim。这个输出不需要复杂资产、材质或引用；它的价值在于让读者第一次看到 Python API 调用如何变成可持久化的 USD scene description。

The expected result is a small ASCII USD layer that can be opened and inspected directly.

常见误读之一是把 `/hello/world` 当成普通目录路径。USD prim path 是 scene namespace 中的路径，它描述 prim 之间的层级关系，不要求磁盘上存在 `hello/world` 目录。另一个误读是把 `xformPrim` 和 `spherePrim` 当成必须继续操作的变量；在本教程中，它们主要证明 `Define` 返回了对应 schema 对象，后续教程才会在这些对象上继续写属性或做查询。

A common misunderstanding is to treat `/hello/world` as a filesystem path rather than a USD scene namespace path.

The page keeps the boundary between `Usd` stage/layer authoring and `UsdGeom` typed geometry schemas intentionally small.

教程步骤主阅读路径 / Tutorial Step Reading Path

  1. 确认环境：先阅读 `Configure your Environment`，保证 Python 能导入 `pxr`。Confirm the environment before running the tutorial script.
  2. 打开脚本：查看 `USD/extras/usd/tutorials/helloWorld/helloWorld.py`，不要翻译或改写其中的 API 名称。Open the provided `helloWorld.py` script and keep API names intact.
  3. 创建 stage：用 `Usd.Stage.CreateNew('HelloWorld.usda')` 建立新的 root layer 输出目标。Create the stage with `Usd.Stage.CreateNew('HelloWorld.usda')`.
  4. 定义父 prim：用 `UsdGeom.Xform.Define(stage, '/hello')` 建立 transform 节点。Define the parent transform prim at `/hello`.
  5. 定义子 prim：用 `UsdGeom.Sphere.Define(stage, '/hello/world')` 建立 sphere 节点。Define the sphere prim at `/hello/world`.
  6. 保存 layer：用 `stage.GetRootLayer().Save()` 写出 `HelloWorld.usda`。Save the root layer to write `HelloWorld.usda`.

关键概念边界 / Concept Boundaries

  • `stage` 不是单纯文件名；它是 OpenUSD 读写 scene description 的编辑入口。A stage is the editing interface for scene description, not merely a file name.
  • `root layer` 才是 `HelloWorld.usda` 这种持久化文件的承载层。The root layer is the persistent layer saved to `HelloWorld.usda`.
  • `/hello/world` 是 USD prim path，不是操作系统路径。`/hello/world` is a USD prim path, not an operating-system path.
  • `Define` 是 authoring API，会创建或确保 typed prim；不要和只读查询 API 混淆。`Define` is an authoring API, not just a query.
  • `Xform` 和 `Sphere` 是 typed schema 的示例；后续教程会扩展到属性、引用、遍历和变体。`Xform` and `Sphere` are typed schema examples used as a basis for later tutorials.
