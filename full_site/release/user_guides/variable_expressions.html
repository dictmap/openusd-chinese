逐段双语理解 / Paragraph-Level Bilingual Coverage

本页说明如何在 layer 的 expressionVariables metadata 中声明变量，并在 asset paths、variant selections、sublayers 等位置用表达式选择值。核心边界是：表达式用于 composition 相关的可变选择，不是通用脚本语言，也不是运行时 Python。This page keeps official API, schema, token, property, function, command, table label, and link semantics unchanged while making the Chinese reading path primary.

阅读路径：先判断页面职责，再看核心 section 和 API/schema/property 分组，随后检查边界、误读点、调试路径和相邻 user guide/schema/API 关系。这样读者可以先靠中文理解主线，再用英文原名核对具体字段。Read purpose first, then sections, grouping, boundaries, diagnostics, and related pages.

工作流 / Workflow

  • 先在 layer metadata 中核对 expressionVariables 的变量名、类型和值，再检查表达式位置是否处在支持变量表达式的字段上。Chinese note first; English/API tokens are retained for verification.
  • 表达式失败时按 defined()、类型匹配、函数返回值、是否允许 None、composition arc 的顺序排查。Chinese note first; English/API tokens are retained for verification.
  • 对 asset path、variant selection 或 sublayer 使用变量时，保留反引号、${VAR}、函数名和路径引号的原始语义。Chinese note first; English/API tokens are retained for verification.

边界与误读点 / Boundaries and common misreads

  • 变量表达式不是脚本语言，不能执行任意逻辑或访问运行时状态；它只提供受限函数和 composition 期求值。Chinese note first; English/API tokens are retained for verification.
  • 不要把 expressionVariables 翻译成中文键名；这是 layer metadata 字段，工具需要按英文名识别。Chinese note first; English/API tokens are retained for verification.
  • None 可能是合法返回，也可能导致目标字段无值；是否安全取决于表达式被用在哪个 USD 字段上。Chinese note first; English/API tokens are retained for verification.
