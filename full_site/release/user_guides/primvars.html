逐段双语理解 / Paragraph-Level Bilingual Coverage

本页解释 primvars 如何在几何体、材质和渲染之间传递 per-primitive、per-face、per-vertex 或 face-varying 数据。核心阅读重点是 interpolation、scene namespace、indexed primvars 和 elementSize，而不是把 primvar 当普通 attribute。This page keeps official API, schema, token, property, function, command, table label, and link semantics unchanged while making the Chinese reading path primary.

官方结构覆盖：本页按 source snapshot 中的主要 section 建立中文解释，覆盖 Primvar Interpolation Modes / Constant Interpolation / Uniform Interpolation / Vertex Interpolation / Varying Interpolation / faceVarying Interpolation / Primvars and the Scene Namespace / Indexed Primvars / Indexed Primvars and Attribute Blocks / Primvar Element Size。这些英文 section 标题用于和官方页逐段核对，不翻译成会破坏链接或锚点的本地标识。Official section headings are preserved for source parity.

阅读路径：先判断页面职责，再看核心 section 和 API/schema/property 分组，随后检查边界、误读点、调试路径和相邻 user guide/schema/API 关系。这样读者可以先靠中文理解主线，再用英文原名核对具体字段。Read purpose first, then sections, grouping, boundaries, diagnostics, and related pages.

调试原则：如果 stage、layer、composition、renderer 或工具结果与预期不同，不要先改 token 或属性名；应从 authored metadata、composition 后值、关系目标、时间采样、材质/渲染上下文和本地链接路径逐层定位。Debug authored data and composed values before changing identifiers.

中文主阅读路径 / Chinese main reading path

本页的晋级目标不是只把 Primvars 放进完成清单，而是让读者在不回到英文正文的情况下理解官方 section 的阅读顺序。中文说明会先给出职责和边界，再把每个 section 连接到实际 authoring、composition、renderer 或工具调试场景。The Chinese path is intended to be readable without relying on the English body.

页面中的 API 名、schema 名、token、属性名、函数名、命令、代码片段、Doxygen 表格标签和链接标题保持英文，是为了让读者能和 OpenUSD 工具、Python/C++ API、usda 文本和官方文档精确对齐。中文解释只负责说明含义、使用顺序和排查方法，不替换正式标识。Identifiers are preserved for technical parity.

对 release user guide 页，section coverage 比单纯字数更重要：每个主要 section 都需要说明它解决什么问题、和哪些相邻概念有关、常见误读是什么、失败时从哪里开始检查。本页把这些信息放在同一个 data-cn-complete 区块内，便于 review_ready_zh 审计判断中文主阅读路径是否足够。Section coverage is the readiness basis.

Official section中文主阅读说明
Primvar Interpolation Modesinterpolation 决定 primvar 值如何映射到几何拓扑。调试错位、材质花纹或属性读取失败时，这是第一层要检查的语义。Official section retained for source parity and local reading continuity.
Constant Interpolationconstant 表示整个 prim 共享一个值，适合对象级颜色、材质选择或整体标记。Official section retained for source parity and local reading continuity.
Uniform Interpolationuniform 常按 face 或 patch 级别变化，适合面级分类、face set 风格数据或低频材质变化。Official section retained for source parity and local reading continuity.
Vertex Interpolationvertex 值随拓扑顶点变化，通常用于点级数据；它不同于 face corner 级别的 faceVarying。Official section retained for source parity and local reading continuity.
Varying Interpolationvarying 表示可以在点之间插值的值，常用于随几何变化但不严格按 face corner 分裂的数据。Official section retained for source parity and local reading continuity.
faceVarying InterpolationfaceVarying 按 face corner 存储，常用于 UV、法线或需要在 seam 处断开的数据。Official section retained for source parity and local reading continuity.
Primvars and the Scene Namespaceprimvars 在 scene namespace 中以约定前缀和属性形式出现，但语义由 UsdGeomPrimvarsAPI 等 API 解释。Official section retained for source parity and local reading continuity.
Indexed Primvarsindexed primvars 把值数组和 index 数组拆开，能减少重复值，但调试时必须同时检查 indices 与 values。Official section retained for source parity and local reading continuity.
Indexed Primvars and Attribute Blocksattribute block 会影响 authored value 的有效性；不要只看默认值，要看 composition 后该时间点是否被 block。Official section retained for source parity and local reading continuity.
Primvar Element SizeelementSize 描述每个元素包含多少个值，尤其影响 shader 或 renderer 如何把数组切片成逻辑元素。Official section retained for source parity and local reading continuity.

工作流 / Workflow

  • 先确认 primvar 名称、typeName、interpolation 和 elementSize，再检查数组长度是否与拓扑数量匹配。Chinese note first; English/API tokens are retained for verification.
  • 材质读取失败时同时检查 UsdGeomPrimvarsAPI、shader primvar reader、material binding 和 renderer 是否支持该 interpolation。Chinese note first; English/API tokens are retained for verification.
  • 如果使用 indexed primvar，调试时同时 dump values、indices 和 authored layer，避免把 index 错误误判为材质错误。Chinese note first; English/API tokens are retained for verification.

边界与误读点 / Boundaries and common misreads

  • primvar 是带几何解释的 attribute，不是任意 custom attribute；它需要和 interpolation、拓扑和 renderer 消费方式一起理解。Chinese note first; English/API tokens are retained for verification.
  • faceVarying 不是 vertex 的同义词；UV seam、法线分裂和 per-corner 数据通常依赖 face-varying 语义。Chinese note first; English/API tokens are retained for verification.
  • indexed primvar 节省存储但增加排查复杂度；indices 越界或长度不匹配会让最终材质结果很难从表面判断。Chinese note first; English/API tokens are retained for verification.
