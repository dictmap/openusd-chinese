逐段双语理解 / Paragraph-Level Bilingual Coverage

`DistantLight` 是 usdLux 中表示远处方向光的 intrinsic light schema。官方定义强调它从 distant source 提供光照，默认沿局部 `-Z axis` 方向照射，也常被称为 `directional light`。中文阅读时不要把它当成场景里有明确近场位置和衰减半径的灯泡；它更像太阳光一类很远、很宽、光线近似平行并影响整个场景的光源。实际朝向通常由 prim 的 transform 决定，因此调试方向时要看局部 `-Z axis` 经 transform 后指向哪里，而不是只看 prim 放在世界空间的哪个位置。

`DistantLight` provides light from a distant source along the local `-Z axis`, and is also known as a directional light.

官方用途段把 `DistantLight` 放在 sunlight 和 distant, broad sources of light 的语境里：光线影响整个 scene，并且大致 parallel，平行程度取决于 `inputs:angle`。这里的关键边界是“远处方向光”与“局部面积光”的区别。`RectLight`、`DiskLight`、`SphereLight` 和 `CylinderLight` 都有具体局部几何；`DistantLight` 的阅读重点则是方向、角直径、强度和阴影软化。它适合做太阳、月光或其他极远光源的主方向照明，不适合表达一个靠近物体、随距离明显衰减的灯具。

It is typically used for broad distant sources such as sunlight, where rays affect the entire scene and are roughly parallel.

官方示例写了 `def DistantLight "Light1"`，把 `float inputs:angle = 1.0`，再把 `float inputs:intensity = 10000`。说明文字指出：示例把 `inputs:angle` 提高到 `1.0`，这会明显拓宽 light spread 并软化 shadow；同时它把强度从默认 `50000` 降到 `10000`。这不是随意调参，而是在展示 `inputs:angle` 和 `inputs:intensity` 的联动：角度变大时光线不再那么平行，阴影更软，场景光量也可能增加，因此需要降低 intensity 或 exposure 来补偿。

The official example increases `inputs:angle` to `1.0` and reduces `inputs:intensity` from the default `50000` to `10000`.

`inputs:angle` 是本页最重要的专有属性之一，USD type 为 `float`，fallback value 为 `0.5299999713897705`。官方说明说它设置 `DistantLight` 的 angular diameter，单位是 degrees；fallback 近似从地球看太阳的 `0.53` 度角大小。角度越高，shadow edges 越柔和，因为 light rays 变得 less parallel；角度越低，阴影越锐利。中文主阅读路径必须把它解释成“远处光源角直径/光线扩散角”，而不是局部灯具的半径、锥角或 spotlight 范围。

`inputs:angle` sets the angular diameter in degrees, with a fallback close to the Sun's angular size as seen from Earth.

官方还给出 `inputs:angle` 的有效范围和性能边界：该值假定在 `0 <= angle < 360`，超出时会被 clipped 到这个范围；如果 `angle > 180`，语义上意味着从超过半球的发光区域贡献远处光。虽然这是有效的，但官方提醒大角度情况下 `DomeLight` 可能提供更好的 performance。也就是说，`DistantLight` 可以表达非常宽的方向光，但如果目标实际上是环境光或 IBL，中文读者应考虑切换到 `DomeLight`，而不是把 `inputs:angle` 一路推高。

The angle is clipped to `0 <= angle < 360`; very large angles may be valid, but `DomeLight` can be a better-performing choice.

官方最后提醒：增加 `inputs:angle` 也会增加 scene 中的 light amount，可能需要降低 light intensity 或 exposure。这个细节很重要，因为很多读者会以为 angle 只改变阴影软硬，不改变总体亮度。排查时应把问题拆开看：如果阴影太硬，检查 `inputs:angle`；如果画面过曝，检查 `inputs:intensity`、`inputs:exposure`、相机 exposure 和色彩管理；如果光照方向不对，检查 transform 和局部 `-Z axis`。不要用一个参数同时承担方向、能量和阴影软化三种职责。

Increasing `inputs:angle` can increase scene light amount, so intensity or exposure may need compensation.

`inputs:intensity` 是线性缩放亮度的属性，USD type 为 `float`，fallback value 为 `50000.0`。官方说明默认 `50000` 提供很高的 intensity，用来近似 sunlight。规范段落把它描述为光发出的 base、unmultiplied luminance，记为 `L`，单位是 nits，也就是 `cd/m²`；公式可写为 `L_Scalar = intensity`。在 RGB renderer 的归一化说明中，`intensity` 为 `1` 且 `exposure` 为 `0` 的可直接可见光通常会在 sensor plane 上生成 `[1, 1, 1]` 像素值，并具有 `1 nit` 亮度；`intensity` 为 `2`、`exposure` 为 `0` 时则对应 `2 nits`。这些英文单位、公式和数值必须保留原样，中文只解释含义。

`inputs:intensity` linearly scales brightness, with fallback `50000.0`, and is described as base luminance in nits.

`light:shaderId` 是 renderer-facing 的 shader 标识，USD type 为 `token`，fallback value 为 `DistantLight`。官方页说明 USD 会注册一个 identifier 为 `DistantLight`、source type 为 `USD` 的 `Sdr` shader node 来对应该灯光的 inputs。这个 token 不应翻译成“远距光”，也不应和 prim name `Light1` 混淆。跨 DCC 或 renderer 交付时，如果灯光没有按 `DistantLight` 被识别，应检查 prim type、`light:shaderId` 是否被覆盖，以及目标 renderer 是否正确解析 usdLux `Sdr` 节点。

`light:shaderId` preserves the `DistantLight` shader identity and corresponding USD `Sdr` node mapping.

继承属性同样会影响阅读和调试。`xformOpOrder` 来自 `Xformable`，决定 transform op 的应用顺序；对于 `DistantLight`，它主要影响局部 `-Z axis` 最终朝向，而不是近场灯位衰减。`proxyPrim`、`purpose` 和 `visibility` 来自 `Imageable`，分别影响代理 prim、渲染用途分类和可见性继承。如果画面没有出现预期光照，不应只看 `inputs:intensity`，还要检查 `visibility` 是否被父层级或 variant 覆盖，`purpose` 是否被渲染管线过滤，layer strength 是否让更强 opinion 覆写了当前属性。

Inherited `Xformable` and `Imageable` properties affect direction, purpose, proxying, and visibility.

还要把“位置”和“方向”的语义分开。很多 DCC 会允许用户移动一个 `DistantLight` prim，但对远处方向光而言，transform 中真正影响照明的是朝向，而不是灯离物体多近。把它移动到场景另一端，通常不应像局部点光或面积光那样改变距离衰减；如果目标是从某个窗口、灯板或发光物体附近照亮角色，`DistantLight` 往往不是合适 schema。中文页必须明确这一点，否则读者容易把太阳光模型误当成可摆放的现场灯具。

For a distant directional light, orientation matters more than local placement or near-field distance.

在 composition 场景中，`inputs:angle` 和 `inputs:intensity` 也要看最终 composed value。弱层写入的 `1.0` 或 `10000` 可能被 stronger layer、variant selection、payload、reference 或 session layer 覆盖；renderer 看到的是组合后的属性，而不是某个源文件中最早出现的 authored value。若本地文档、usdview 和最终渲染结果不一致，应先用属性面板、`usdcat` 或 `usdtree` 确认最终 stage 上的取值，再判断是文档理解、资产层级还是 renderer 行为导致差异。

Authored values must be interpreted through USD composition to find the stage's composed values.

建议调试顺序是：先确认 prim type 是 `DistantLight`，再确认 transform 后的局部 `-Z axis` 是否指向目标照明方向；接着检查 `inputs:angle` 是否表达期望的太阳角/阴影软化，`inputs:intensity` 与 exposure 是否匹配；然后检查 `visibility`、`purpose`、payload、variant、composition 和 stronger layer opinion；最后再看 renderer 对 distant light、spectral radiance、shadow sampling 和 color management 的实现。这样可以避免把方向错误、阴影太硬、曝光过强、层级覆盖和 renderer 行为混成一个问题。

A practical debugging path is schema type, transform direction, angle, intensity/exposure, visibility/composition, then renderer support.

和相邻 usdLux 页面一起读时，`DistantLight` 的职责边界很清楚：`DiskLight` 是圆盘面积光，`RectLight` 是矩形面积光，`SphereLight` 是球形或近似点状局部光，`CylinderLight` 是管状侧面发光，`DomeLight` 更适合环境或图像照明，`PortalLight` 帮助把环境采样引导到开口区域，`ShapingAPI` 和 `ShadowAPI` 则补充塑形和阴影控制。`DistantLight` 只负责远处方向光，不应拿来表达局部灯具外观、环境贴图或 portal 采样策略。

Adjacent usdLux pages separate distant directional illumination from disk, rectangular, spherical, cylindrical, dome, portal, shaping, and shadow concepts.

USDA 示例阅读 / USDA Example Reading

下面保留官方示例中的关键灯光结构。代码标识符保持英文原样；中文只解释职责和调参关系。

#usda 1.0
(
    upAxis = "Y"
)

def Scope "Lights"
{
    def DistantLight "Light1"
    {
        float inputs:angle = 1.0
        float inputs:intensity = 10000
    }
}
  • `def DistantLight "Light1"` 表示 prim type 是 `DistantLight`，`Light1` 只是 prim name。The prim type is `DistantLight`; `Light1` is the prim name.
  • `inputs:angle = 1.0` 会拓宽 light spread，并软化 shadow edges。A larger angle broadens the spread and softens shadow edges.
  • `inputs:intensity = 10000` 是示例中的补偿选择，低于默认 `50000.0`。The example reduces intensity from the default `50000.0`.
