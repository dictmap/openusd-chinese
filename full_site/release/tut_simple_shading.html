范围说明 / Scope Note

本页是 OpenUSD release 教程 Simple Shading in USD 的完整双语复刻页。中文放在主阅读路径前面，用来解释官方教程的目标、步骤、API 边界和常见误读；英文页面名、API 名、类名、shader id、token、代码片段、命令和链接保持原样，便于和官方文档核对。

This page is a Chinese-first bilingual reproduction of the OpenUSD release tutorial Simple Shading in USD. English identifiers, API names, shader ids, tokens, commands, code snippets, and links are retained for technical comparison.

官方教程演示如何用 Python USD API 创建一个简单资产：先建立 Model 根节点和一张带 st 纹理坐标的 quadrilateral billboard，再创建 UsdShade.Material、UsdPreviewSurface、UsdUVTexture 与 UsdPrimvarReader_float2 组成的 shading network，最后通过 UsdShade.MaterialBindingAPI 把几何体绑定到材质。

The official tutorial creates a model asset, authors a textured billboard mesh, builds a UsdShade material network, and binds the material to geometry.

逐段双语理解 / Paragraph-Level Bilingual Coverage

这篇教程的核心不是“给一个 mesh 贴图”这么简单，而是说明 USD 如何把资产组织、几何数据、材质容器、shader 网络、纹理坐标和材质绑定分成不同层次。读者应先抓住这条主线：Usd.Stage 保存场景，UsdGeom.Xform 作为模型根，UsdGeom.Mesh 负责几何拓扑和 primvars，UsdShade.Material 负责容纳 shading network，UsdShade.Shader 节点通过 input/output 连接表达数据流，MaterialBindingAPI 只负责说明哪个几何体使用哪个材质。

The key idea is the separation between scene organization, geometry, material containers, shader networks, texture coordinates, and material binding.

开头的 Making a Model 段落用 Usd.Stage.CreateNew("simpleShading.usda") 创建文本层，并用 UsdGeom.SetStageUpAxis(stage, UsdGeom.Tokens.y) 写入 stage 级坐标约定。随后 UsdGeom.Xform.Define(stage, "/TexModel") 建立模型根，Usd.ModelAPI(modelRoot).SetKind(Kind.Tokens.component) 把它标记为 component。这个根节点的意义在于把几何和材质一起包装成一个可引用的资产单元，而不是把 Mesh 和 Material 随便散落在 root scope 下。

The model root collects geometry and shading prims under one referencable asset root rather than leaving them as unrelated root-scope prims.

接下来的 Adding a Mesh "Billboard" 段落创建 /TexModel/card 这个四边形 UsdGeom.Mesh。示例写入 points、faceVertexCounts、faceVertexIndices 和 extent，使 billboard 有明确的拓扑和包围范围。它没有显式创建 normals，因为默认 mesh 采用 Catmull-Clark subdivision，渲染器可以解析出法线；如果真实资产需要多边形显示而非细分表面，应另外考虑 CreateSubdivisionSchemeAttr() 等设置。

The billboard mesh authors topology and extent. Normals are omitted because the default subdivision scheme can provide analytic normals to renderers.

同一段里的 UsdGeom.PrimvarsAPI(billboard).CreatePrimvar("st", Sdf.ValueTypeNames.TexCoord2fArray, UsdGeom.Tokens.varying) 是理解贴图的关键。st 是几何体上的纹理坐标 primvar，值把四个角映射到 unit square 的 UV 空间。这里的 st 名称不是中文术语，而是 shading network 后续会读取的 token；如果材质网络里的 primvar reader 读的名字和几何上的 primvar 名字不一致，纹理通常会显示错误或使用 fallback。

The st primvar stores texture coordinates on the geometry. The shader network must read the same primvar name to map the texture correctly.

Make a Material 段落用 UsdShade.Material.Define(stage, "/TexModel/boardMat") 创建材质 prim。Material 在 USD 中不是直接的颜色属性，也不是贴图文件路径；它是一个容器，可以包含 surface、displacement、renderer-specific 等不同用途的 shader network。复杂资产会有多个 Gprim 绑定到不同 Material，而这个教程只有一张 billboard，所以只需要一个 boardMat。

A Material is a container for shading networks, not just a color or a texture path.

Add a UsdPreviewSurface 段落创建 /TexModel/boardMat/PBRShader，并把 shader id 设为 UsdPreviewSurface。这个 id 要保持英文原样，因为它是被 Hydra、DCC 和 renderer delegate 识别的 shader 标识。示例设置 roughness 和 metallic input，然后用 material.CreateSurfaceOutput().ConnectToSource(pbrShader.ConnectableAPI(), "surface") 把材质的 surface output 连接到 shader 的 surface output。这个连接决定材质表面的最终 shading source。

UsdPreviewSurface is the preview surface shader id. The material surface output is connected to the shader surface output to identify the final surface shading source.

Add Texturing 段落添加两个 shader 节点：UsdUVTexture 负责读取并采样图像文件，UsdPrimvarReader_float2 负责从被绑定的几何体上读取 float2 类型的纹理坐标。两者的关系是：primvar reader 输出 result，连接到 texture shader 的 st input；texture shader 输出 rgb，连接到 UsdPreviewSurface 的 diffuseColor input。这样图像颜色才会成为 surface diffuse color 的来源。

UsdPrimvarReader_float2 provides texture coordinates to UsdUVTexture, and the texture rgb output feeds the preview surface diffuseColor input.

diffuseTextureSampler.CreateInput("file", Sdf.ValueTypeNames.Asset).Set("USDLogoLrg.png") 说明纹理路径是 asset-valued input，会经过 USD 的 asset resolution 机制解释。它不是普通 Python 字符串路径的最终语义。打包资产或跨机器运行时，要确保 USDLogoLrg.png 与 layer 的相对路径、resolver context 或 usdz 包内布局一致，否则材质网络虽然存在，图像仍可能找不到。

The texture file input is an asset-valued path. It must resolve correctly for the texture to appear.

教程没有把 varname 直接写死在 stReader 上，而是创建 material.CreateInput("frame:stPrimvarName", Sdf.ValueTypeNames.Token) 并把它设为 st，再让 stReader.CreateInput("varname", ...).ConnectToSource(stInput) 连接到这个 material input。这样做的教学意义很重要：材质可以把内部网络深处的参数暴露成公共接口，资产使用方不用进入 shader 子树，也能看到或覆盖材质的关键控制项。

The material input exposes a deep shader-network parameter as a public interface, making the primvar name easier to inspect and override.

最后的 MaterialBindingAPI 步骤表达“这个几何体使用这个材质”。示例先在 billboard prim 上应用 UsdShade.MaterialBindingAPI，再执行 UsdShade.MaterialBindingAPI(billboard).Bind(material)。绑定不是把材质数据复制到 mesh 上，也不是修改 face topology；它在 scene description 中写入关系，让材质解析过程能从 gprim 找到对应的 Material prim。后续更复杂的场景还会涉及 collection binding、binding strength 和 material purpose，本教程只展示最直接的 per-prim binding。

Material binding authors a relationship from the geometry to the material. It does not copy shader data onto the mesh.

运行路径上，教程把脚本和数据放在 USD 发行包的 USD/extras/usd/tutorials/simpleShading 中，聚合脚本为 generate_simpleShading.py，最终产物是 simpleShading.usda。阅读时不要只看 Python 片段，也要看保存后的 usda 文本：那里能看到 def Material、def Shader、inputs:、outputs:、连接关系、rel material:binding 以及 primvars:st 如何落到场景描述里。

The Python examples author a simpleShading.usda layer. Inspecting the resulting text layer shows the material, shaders, inputs, outputs, connections, binding relationship, and primvars.

常见误读有四个。第一，UsdPreviewSurface 是可移植的预览材质模型，不等于某个生产渲染器的完整 shader 语言。第二，st 是几何 primvar 与 shader reader 之间的约定名，不是自动出现的 UV。第三，Material 是网络容器，真正的 shader 参数在其子节点上或暴露为 material input。第四，Bind(material) 只是绑定关系，显示结果还取决于 renderer 是否支持这些 preview shader 节点、纹理路径是否能解析、以及几何是否提供了匹配的 texture coordinates。

Common pitfalls include confusing preview shading with renderer-specific shaders, assuming st appears automatically, treating Material as a flat property bag, or forgetting that binding only identifies the material.

从 section 覆盖角度看，本教程把 USD 着色分成三条可检查的证据链。第一条是 scene description 证据：simpleShading.usda 中应该能看到 TexModel、card、boardMat、PBRShader、diffuseTexture 和 stReader 这些 prim。第二条是 connection 证据：outputs:surface、inputs:diffuseColor、inputs:st、outputs:rgb 和 outputs:result 之间应形成方向明确的数据流。第三条是 binding 证据：card 或其合适祖先上应有 material:binding 关系指向 boardMat。三条证据同时存在，才说明材质网络既被写入，又被几何实际使用。

A reliable inspection path checks authored prims, shader connections, and the material binding relationship together.

阅读保存后的 usda 时，可以把它看成对 Python authoring 代码的结构化回放。def Material 说明材质容器存在；def Shader 说明网络节点存在；uniform token info:id 说明节点类型可被渲染系统识别；inputs 和 outputs 说明可连接端口；connect 关系说明数据从一个 shader 输出流向另一个 shader 输入；rel material:binding 则说明几何选择了哪个 material。只看 viewport 中“有没有图”是不够的，因为图像可能因为 fallback、缓存或 renderer 默认材质而误导判断。

The saved usda layer is the audit trail for authoring decisions and should be inspected alongside viewport results.

本页还保留一个重要边界：UsdShade 描述的是跨工具可交换的 shading graph 结构，而不是保证每个 renderer 都给出完全相同的最终像素。UsdPreviewSurface 的价值是提供稳定的预览语义，让 usdview、Hydra delegate 和 DCC 插件可以用共同的最低语义理解材质。生产渲染中可以继续扩展 renderer-specific 节点，但在教程层面应先确认 preview network、texture asset、primvar reader 和 material binding 这四个基础环节可靠。

UsdShade describes portable shading structure; renderer-specific fidelity is a separate layer above the preview network.

把这个教程迁移到真实资产时，建议保持类似组织方式：资产根下放几何和 Looks 或材质 scope；几何只声明拓扑、extent 和 primvars；材质下放 shader 节点；需要调参的内部 shader input 暴露到 material input；最后用 binding 明确指向 material。这样做的好处是下游工具能替换纹理、重定向 primvar、覆盖材质参数或重新绑定材质，而不必破坏 mesh 拓扑。

The pattern scales to real assets by separating geometry, material interface, shader internals, and binding.

因此，本教程的中文主阅读路径可以按“建资产根、建 billboard、写 st、建 Material、接 preview surface、接 texture sampler、接 primvar reader、暴露 varname、绑定材质、保存并检查 usda”这十步执行。每一步都有可见的 USD spec 或 API 调用与之对应；如果某一步缺失，问题通常会落在路径解析、token 名称、连接方向、材质绑定位置或 renderer 支持范围上。

The practical reading path is a ten-step checklist from asset root creation through saved-layer inspection.

最后做人工复核时，应同时打开脚本、生成的 usda 文本和 usdview 结果：脚本验证 authoring 意图，usda 验证已写入的 scene description，usdview 验证常见预览渲染路径。三者一致，才算这个简单着色示例真正可复现，并能作为后续复杂材质网络的最小检查基线。

Manual review should compare the script, the saved layer, and the usdview result.

教程步骤 / Tutorial Steps

  1. 创建 stage 和模型根：用 Usd.Stage.CreateNew、UsdGeom.SetStageUpAxis、UsdGeom.Xform.Define 和 Usd.ModelAPI.SetKind 建立一个可引用的 component 资产根。Create the stage and model root as a component asset.
  2. 创建 billboard mesh：写入四个点、一个四边形 face、extent，以及名为 st 的 TexCoord2fArray primvar。Create the billboard mesh and its st texture-coordinate primvar.
  3. 创建 UsdShade.Material：把材质放在 /TexModel/boardMat 下，使 shading network 与资产根保持同一组织边界。Create a material container under the model.
  4. 添加 UsdPreviewSurface：创建 PBRShader，设置 roughness 和 metallic，并连接 material 的 surface output。Add the preview surface shader and connect it as the material surface.
  5. 添加 texture 和 primvar reader：让 UsdPrimvarReader_float2 提供 st 坐标，让 UsdUVTexture 采样 USDLogoLrg.png，再把 rgb 连到 diffuseColor。Wire the primvar reader, texture sampler, and diffuse color input.
  6. 公开 primvar 名称并绑定材质：通过 material input 暴露 frame:stPrimvarName，再用 MaterialBindingAPI 把 billboard 与 boardMat 关联起来并保存 layer。Expose the primvar name on the material interface, bind the material, and save the layer.

关键 API 与语义边界 / API Boundaries

UsdGeom.Mesh 负责几何形状、拓扑、extent 和 primvars；它不保存材质网络。

UsdGeom.Mesh owns geometry and primvars, not the shader graph.

UsdShade.Material 是 shader network 的容器和公共接口边界；它可以暴露内部 shader input。

UsdShade.Material is the container and public interface for shading networks.

UsdPreviewSurface 是预览 surface shader id；保留英文 token 是互操作要求。

UsdPreviewSurface is the portable preview surface shader id.

UsdUVTexture 读取 asset-valued texture file，并根据 st input 输出 rgb 等结果。

UsdUVTexture samples the texture file using st coordinates.

UsdPrimvarReader_float2 从绑定几何上读取 float2 primvar；本例读取 st。

UsdPrimvarReader_float2 reads a float2 primvar such as st.

MaterialBindingAPI 写入材质绑定关系；它不是渲染，也不是贴图采样。

MaterialBindingAPI authors the binding relationship.

核心代码摘录 / Core Code Fragments

下面保留精简代码片段，用于核对官方步骤。它们不是逐行重印官方教程，而是标出本页中文解释中的关键 API 连接点。

These fragments highlight the API connection points discussed above.

from pxr import Gf, Kind, Sdf, Usd, UsdGeom, UsdShade

stage = Usd.Stage.CreateNew("simpleShading.usda")
UsdGeom.SetStageUpAxis(stage, UsdGeom.Tokens.y)
modelRoot = UsdGeom.Xform.Define(stage, "/TexModel")
Usd.ModelAPI(modelRoot).SetKind(Kind.Tokens.component)

billboard = UsdGeom.Mesh.Define(stage, "/TexModel/card")
texCoords = UsdGeom.PrimvarsAPI(billboard).CreatePrimvar(
    "st", Sdf.ValueTypeNames.TexCoord2fArray, UsdGeom.Tokens.varying)

material = UsdShade.Material.Define(stage, "/TexModel/boardMat")
pbrShader = UsdShade.Shader.Define(stage, "/TexModel/boardMat/PBRShader")
pbrShader.CreateIdAttr("UsdPreviewSurface")
material.CreateSurfaceOutput().ConnectToSource(pbrShader.ConnectableAPI(), "surface")
stReader = UsdShade.Shader.Define(stage, "/TexModel/boardMat/stReader")
stReader.CreateIdAttr("UsdPrimvarReader_float2")

diffuseTextureSampler = UsdShade.Shader.Define(stage, "/TexModel/boardMat/diffuseTexture")
diffuseTextureSampler.CreateIdAttr("UsdUVTexture")
diffuseTextureSampler.CreateInput("file", Sdf.ValueTypeNames.Asset).Set("USDLogoLrg.png")
diffuseTextureSampler.CreateInput("st", Sdf.ValueTypeNames.Float2).ConnectToSource(
    stReader.ConnectableAPI(), "result")
pbrShader.CreateInput("diffuseColor", Sdf.ValueTypeNames.Color3f).ConnectToSource(
    diffuseTextureSampler.ConnectableAPI(), "rgb")

stInput = material.CreateInput("frame:stPrimvarName", Sdf.ValueTypeNames.Token)
stInput.Set("st")
stReader.CreateInput("varname", Sdf.ValueTypeNames.Token).ConnectToSource(stInput)

billboard.GetPrim().ApplyAPI(UsdShade.MaterialBindingAPI)
UsdShade.MaterialBindingAPI(billboard).Bind(material)
stage.Save()

调试与常见误读 / Debugging Notes

  • 如果 mesh 可见但没有纹理，先检查 USDLogoLrg.png 是否能通过当前 layer 的 asset path 解析到，再检查 renderer 是否支持 UsdPreviewSurface 和 UsdUVTexture。If the mesh appears without the texture, check asset resolution and preview shader support first.
  • 如果纹理坐标异常，检查几何上是否真的有 primvars:st，以及 UsdPrimvarReader_float2.inputs:varname 是否通过 material interface 指向同一个 st token。Texture-coordinate issues usually point to a missing or mismatched st primvar.
  • 如果材质完全没有作用，检查 rel material:binding 是否写在实际 gprim 或它的合适祖先上，并确认绑定目标路径指向 def Material 而不是 shader 子节点。A missing effect may mean the material binding relationship is absent or points to the wrong prim.
  • 不要把保存后的 simpleShading.usda flatten 成唯一诊断来源。flatten 能展示最终组成结果，但调试 authoring 时仍要看 root layer 中的原始 material、shader、connection 和 binding specs。Flattened output is useful, but authoring diagnostics still need the original layer specs.
  • 不要翻译 UsdPreviewSurface、UsdUVTexture、UsdPrimvarReader_float2、diffuseColor、surface、rgb、st 等标识符；它们是技术互操作点，不是普通正文。Do not translate identifiers and token literals that define the shader graph semantics.
