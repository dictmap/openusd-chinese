逐段双语理解 / Paragraph-Level Bilingual Coverage

本页讨论如何在 USD 中记录和绑定命名坐标系统，让 shader、projection、camera 和材质网络能引用稳定的 frame of reference。中文主阅读路径应覆盖命名、scope、多坐标系统、API schema、frame 记录、binding 和消费端分析。读者需要把 CoordSysAPI 理解为场景描述中的参考系声明和绑定机制，而不是把它当作直接修改 prim transform 的工具。Chinese is the main reading path; English identifiers, API names, schema names, tokens, property names, code, commands, and links are retained for verification.

Purpose Purpose 说明需求来自 shading 和 projection：shader 需要知道某个名称对应的坐标系在哪里、如何绑定、如何在层级中解析。没有显式机制时，不同工具会用私有约定解释 reference frame。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Requirements 要求包括按名称识别、必须有作用域、一个 prim 可以拥有多个坐标系统，以及 shader 能通过名称引用。中文解释要强调名称不是随意文本，而是 authoring 和消费端都必须遵守的协议。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Coordinate Systems are Identified by Name in Shaders shader 通过名称引用坐标系统，因此名称、scope 和绑定路径必须一致。调试 shader 投影错误时，先检查名称是否匹配，再检查绑定目标和 transform。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Coordinate Systems Must be Scoped scope 约束避免全局名称冲突。局部作用域能让不同资产或子树使用相同名称而不互相污染，这对于引用、payload 和资产复用尤其重要。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Proposed API Schema Proposed API Schema 讨论如何用 API schema 记录坐标系。CoordSysAPI 名称、属性和关系必须保留英文；中文应解释它们如何在 prim 上 author 和解析。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Recording a Frame of Reference 记录 frame of reference 的重点是把某个 prim 或变换作为坐标系定义来源。它描述参考系，而不是替代 xformOp 或更改几何本身。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Binding Frames of Reference binding 把 shader 或消费端需要的坐标系统名称连接到具体 frame。绑定错误通常表现为投影偏移、纹理空间错位或 renderer 无法找到坐标系统。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
CoordSysAPI CoordSysAPI 是核心 API schema，负责声明和绑定坐标系统。它与 UsdGeomXformable、shader 输入和材质网络相邻，但职责不同。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
USD Sample and Analysis 示例和分析展示坐标系统如何被 author、绑定和消费。阅读时应保留代码、属性名和 schema 名，用中文解释每一步的引用关系。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.

中文主阅读路线 / Chinese Main Reading Path

  1. 调试投影或 procedural texture 错位时，先确认 shader 期望的 coordinate system 名称，再检查 CoordSysAPI 是否在正确 scope 中声明和绑定。Operational reading and debugging path.
  2. 资产被 reference 或 payload 到更大 stage 后，如果坐标系统失效，应检查 scope 是否仍可解析，避免全局名称假设被打破。Operational reading and debugging path.
  3. 需要 camera projection 或局部 reference frame 时，把本页作为设计说明，再回到当前 schema/API 页核对具体 authoring 方法。Operational reading and debugging path.

边界、误读点与调试路径 / Boundaries, Misreads, Debugging

  • CoordSysAPI 不直接移动模型，它记录和绑定 reference frame；实际几何变换仍由 xform stack 表达。Boundary or common misread to avoid.
  • 坐标系统名称不能随意翻译或改写，因为 shader 和消费端按名称解析。Boundary or common misread to avoid.
  • 本页讨论提案语义，具体实现状态和 API 细节仍需与当前 OpenUSD 版本核对。Boundary or common misread to avoid.
