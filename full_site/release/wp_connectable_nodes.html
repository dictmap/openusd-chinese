逐段双语理解 / Paragraph-Level Bilingual Coverage

本页讨论把 UsdShade 中成熟的 connectable node 机制推广到其他域，例如 UsdLux 和 UsdRi。中文阅读重点是理解 node definition、API schema、plugin-defined behavior、connectability rules 和 Sdr/Ndr 之间的关系。它不是让所有网络都变成 shader network，而是抽象出输入输出连接、类型规则和插件回调，使非 shading 网络也能获得一致的连接语义。Chinese is the main reading path; English identifiers, API names, schema names, tokens, property names, code, commands, and links are retained for verification.

中文主阅读路线 / Chinese Main Reading Path

  1. 设计新的节点型 schema 时，先判断是否需要 input/output 连接，再决定是否采用 ConnectableAPI 语义，而不是直接复制 UsdShade 类名。Operational reading and debugging path.
  2. 调试连接失败时，检查连接目标、输入输出类型、plugin-defined connectability behavior 和 Sdr/Ndr 定义是否一致。Operational reading and debugging path.
  3. 扩展到 UsdLux 或 renderer-specific 节点时，保留 UsdShade 的已有规则，同时明确新域自己的连接约束。Operational reading and debugging path.

边界、误读点与调试路径 / Boundaries, Misreads, Debugging

  • connectable node 泛化不是把所有网络都变成材质网络；它只是复用连接和验证机制。Boundary or common misread to avoid.
  • ConnectableAPI、UsdShade、Sdr、Ndr、UsdLux、UsdRi 等正式标识必须原样保留。Boundary or common misread to avoid.
  • 插件回调提供灵活性，也带来一致性风险；没有清晰规则的连接会让工具链难以验证。Boundary or common misread to avoid.
