范围说明 / Scope Note

本页是 OpenUSD release 教程中“遍历 stage”的完整双语参考页。中文主线解释如何在 `usdview` interpreter 中用 `usdviewApi.stage.Traverse()` 遍历 composed prims，如何用 Python generator 过滤 schema，何时改用 `Usd.PrimRange.PreAndPostVisit`，以及 `active`、`defined`、`loaded`、`concrete` predicate 对遍历结果的影响。英文 API 名称、文件名、prim path、代码和官方链接保留，用于和官方教程核对。

This local bilingual page explains the OpenUSD tutorial for traversing a stage in Chinese first while retaining official API names, file names, prim paths, code, and links for comparison.

逐段双语理解 / Paragraph-Level Bilingual Coverage

本教程讨论的是在 `UsdStage` 上遍历已经 composition 完成的 prim，而不是逐行扫描某个 `.usda` 文件。它使用上一页 `Referencing Layers` 生成的两个 layer：被引用的 `HelloWorld.usda` 和引用方 `RefExample.usda`。当你打开 `RefExample.usda` 时，stage 视图会把 reference arc、local override 和 session layer 等意见组合成一个可访问的 scenegraph；遍历 API 看到的是这个 composed view。

The tutorial traverses composed prims on a `UsdStage`, using the layers produced by the referencing tutorial.

第一步是在 `usdview` 中打开 `RefExample.usda`，再按 `i` 或通过 `Window > Interpreter` 打开 interpreter。这个 interpreter 里有内置对象 `usdviewApi`，它提供当前 `usdview` 会话和已打开 `UsdStage` 的便利入口。示例里的 `usdviewApi.stage` 就是当前 stage；不要把它翻译成普通“舞台变量”，也不要把它和脚本自己创建的 `Usd.Stage.Open(...)` 混淆。

The `usdview` interpreter exposes a `usdviewApi` object whose `stage` member is the open `UsdStage`.

最简单的遍历入口是 `usdviewApi.stage.Traverse()`。它返回一个 Python generator，按 depth-first traversal 顺序产生 prim。官方示例 `[x for x in usdviewApi.stage.Traverse()]` 会得到 `Usd.Prim()`、`Usd.Prim()`、`Usd.Prim()`、`Usd.Prim()`。这些路径来自 composed stage：`/refSphere/world` 是引用后映射出来的子 prim，不是源 layer 中的 `/hello/world` 原样路径。

`Usd.Stage.Traverse()` yields prims from the composed stage in depth-first order.

因为 `Traverse()` 是 generator，所以可以像普通 Python 迭代器一样过滤。示例 `[x for x in usdviewApi.stage.Traverse() if UsdGeom.Sphere(x)]` 只保留可被 `UsdGeom.Sphere` schema 包装的 prim，结果是 `` 和 ``。这个写法非常接近实际 importer 或 imaging client 的结构：先遍历 stage，再按 schema、typeName、path、purpose 或自定义 metadata 选择要处理的 prim。

Traversal results can be filtered with normal Python generator expressions, for example by checking `UsdGeom.Sphere(x)`.

更复杂的遍历要用 `Usd.PrimRange()`。官方示例 `Usd.PrimRange.PreAndPostVisit(usdviewApi.stage.GetPseudoRoot())` 会产生 pre-order 和 post-order 两次访问，并可用 `primIter.IsPostVisit()` 区分当前事件。pre visit 适合在进入子树前建立状态，post visit 适合在离开子树后汇总或弹栈。渲染、导入、层级统计、父子继承上下文处理，都可能需要这种“进入/退出”成对访问。

`Usd.PrimRange.PreAndPostVisit` exposes pre-order and post-order visitations for more involved traversal logic.

`usdviewApi.stage.GetPseudoRoot()` 是遍历整棵 stage 的常见起点。pseudo-root 不是一个普通 scene prim，也不会对应用户资产中的真实对象；它是 stage namespace 的虚拟根，路径显示为 ``。从 pseudo-root 开始可以覆盖所有根 prim，例如 `/refSphere` 与 `/refSphere2`。如果只从某个 prim 开始，则遍历范围会限制在该 prim 的子树内。

The pseudo-root is a virtual root used to start traversal over the whole stage namespace.

本页还解释了 prim-flag predicates。`Usd.Stage.Traverse()` 实际上是便利方法：它对 composed scenegraph 中 active、defined、loaded、concrete 的 prim 做 pre-order traversal。`defined` 与 prim 是否由 `def` 支撑有关，区别于只提供 weaker override 的 `over`；`loaded` 与 payload 加载边界相关；`concrete` 与 class 等抽象 prim 相关；`active` 则决定 prim 及其 namespace children 是否在 composed scenegraph 中有存在感。

`Stage.Traverse()` applies prim flags for active, defined, loaded, and concrete prims.

教程接着在 `usdview` 中选择 `refSphere2` 并执行 `Edit > Deactivate`。这个操作不会修改资产 layer，而是在 session layer 中 author 一个临时意见：`over "refSphere2" ( active = false )`。session layer 用于当前工作会话中不希望提交给资产的意见，例如临时隐藏、选择、调试和交互式编辑。理解这一点可以避免把 usdview 中的临时操作误认为已经改动了 `RefExample.usda`。

Deactivating `refSphere2` in `usdview` authors a transient `active = false` opinion in the session layer.

deactivate 后，`Traverse()` 不再访问 `refSphere2` 或它的 namespace children，渲染器也不会绘制它。官方示例再次执行 `[x for x in usdviewApi.stage.Traverse()]`，只剩 `` 和 ``。这说明 `active` predicate 不是简单 UI 过滤，而是影响 composed scenegraph 存在性的语义：inactive prim 可被 stage 知道，但其子树不再作为 composed prims 出现。

After deactivation, `Traverse()` skips `refSphere2` and its children because inactive prims do not contribute to the composed scenegraph in the same way.

为了验证 inactive prim 仍然可被更底层地观察，教程使用 `usdviewApi.stage.TraverseAll()`。`TraverseAll()` 不应用默认 predicates，因此可以看到 `Usd.Prim()`，但仍看不到 ``，因为 inactive prim 的 namespace children 已经没有 composed presence。这个例子是理解 `Traverse()` 与 `TraverseAll()` 差异的关键：前者面向正常消费者，后者适合诊断和检查被过滤的 prim 状态。

`TraverseAll()` applies no predicates, making it useful for inspecting prims that default traversal would skip.

从工具开发角度看，本教程给出了遍历 stage 的最小模式：先选择 traversal 范围，再决定 predicate，再按 schema 或类型过滤，最后对匹配 prim 执行业务逻辑。imaging client 会把匹配 prim 转成渲染对象；DCC importer 会把它们转成目标应用的数据结构；批处理检查器会读取属性并报告问题。遍历本身只是访问框架，真正的业务判断通常发生在遍历回调或列表推导的过滤条件中。

A traversal pipeline usually chooses a range, applies predicates, filters by schema or type, and then performs client-specific work.

常见误读有三类。第一，把 `Traverse()` 看成文件顺序遍历；实际上它遍历 composed stage。第二，把 inactive prim 看成完全删除；实际上 session layer 或其他强层可能只是 author 了 `active=false`，不同遍历 API 会暴露不同视图。第三，把 `PrimRange.PreAndPostVisit` 当成普通列表；实际上它携带进入和退出子树的事件，适合需要维护栈状态的算法。

Common mistakes include confusing stage traversal with file traversal, deactivation with deletion, and pre/post visitation with a flat list.

维护本页时必须保留这些字面量：`RefExample.usda`、`HelloWorld.usda`、`USD/extras/usd/tutorials/traversingStage`、`usdviewApi`、`usdviewApi.stage.Traverse()`、`UsdGeom.Sphere(x)`、`Usd.PrimRange.PreAndPostVisit`、`GetPseudoRoot()`、`primIter.IsPostVisit()`、`GetSessionLayer().ExportToString()`、`active = false`、`TraverseAll()`、`/refSphere`、`/refSphere/world`、`/refSphere2` 和 `/refSphere2/world`。这些名称是读者复现实验和查 API 的锚点，不应翻译。

The identifiers, file names, API calls, and prim paths are the reproducibility anchors for this tutorial.

对 `defined`、`loaded`、`concrete` 三个 predicate 也要建立边界。`defined` 关注 prim 是否由 `def` 等定义性意见支撑，而不是只由 `over` 提供覆盖；`loaded` 关注 payload 边界是否已经载入，未载入 payload 的内容不会像普通子树一样进入默认遍历；`concrete` 用来排除 class 这类抽象 prim。教程没有深入 payload 和 class，但把这些词放在这里，是为了说明 `Traverse()` 的默认结果已经替你做了消费者通常想要的过滤。

The default traversal predicates filter by defined, loaded, concrete, and active state.

如果正在写导入器或渲染前端，通常不要直接使用 `TraverseAll()` 作为主循环。`TraverseAll()` 会暴露诊断视图中的 prim，可能包括 inactive 或不适合普通消费的对象；这对排查问题有用，但对构建最终渲染列表或 DCC 节点列表可能过宽。主循环先用 `Traverse()`，只有在需要解释“为什么某个 prim 没被处理”时再切到 `TraverseAll()` 或自定义 `Usd.PrimRange`，是更稳的工程策略。

Consumers usually start with `Traverse()` and reserve `TraverseAll()` for diagnostics or custom traversal behavior.

本页还把 session layer 引入了实际调试语境。session layer 的意见比普通 root layer 或 sublayer 更适合临时交互，它可以改变当前 stage 视图，却不污染资产文件。`usdview` 中 deactivate 一个 prim 后马上影响 `Traverse()`，正好说明遍历结果不是“文件里有什么”的机械反映，而是“当前 stage 所有强弱意见解析后，消费者应该看见什么”的结果。

Session-layer deactivation demonstrates that traversal reflects the current composed stage view.

在真实工具中，遍历通常还会和自定义 predicate 或显式 `Usd.PrimRange` 组合使用。比如资产检查器可能需要看到 inactive prim 来报告配置问题，渲染前端通常只需要 active、loaded、concrete 的可消费 prim，导入器则可能按 `IsA`、`GetTypeName()`、`GetPath()`、purpose 或自定义 metadata 分组处理。选择哪种遍历不是性能细节，而是决定工具语义的第一步：你是在模拟最终消费者看到的场景，还是在诊断所有潜在 scene description？本页用 `Traverse()` 与 `TraverseAll()` 的对比，把这个边界最小化地展示出来。

Real tools combine traversal with predicates and schema/type filters depending on whether they consume or diagnose scene description.

前后序访问同样有实际用途。进入 prim 时可以继承父级状态，例如当前 model、material binding、purpose、visibility 或变换栈；离开 prim 时可以清理状态、汇总子树统计，或把子节点结果归并到父节点。只需要平铺列表时 `Traverse()` 足够；一旦算法依赖父子进入/退出边界，就应考虑 `Usd.PrimRange.PreAndPostVisit`，并用 `IsPostVisit()` 明确区分两次访问。

Pre/post visitation is useful when traversal logic needs inherited state or subtree aggregation.

本页的中文可读目标是让读者不用依赖英文正文也能回答：`Traverse()` 默认遍历哪些 prim、为什么 `UsdGeom.Sphere(x)` 可以过滤 sphere、pre/post visit 适合什么算法、deactivate 为什么写入 session layer、`TraverseAll()` 为什么能看到 inactive prim 但看不到它的 children。能回答这些问题，读者就可以把 traversal 模式迁移到 importer、检查器、渲染前端和资产分析脚本。

The reading goal is that the reader can explain traversal predicates, schema filtering, pre/post visits, session-layer deactivation, and `TraverseAll()` behavior.

教程步骤 / Tutorial Steps

  1. 准备文件：复制 `USD/extras/usd/tutorials/traversingStage`，确保 `HelloWorld.usda` 与 `RefExample.usda` 可写。Copy the tutorial folder and keep the USD files writable.
  2. 打开 stage：在 `usdview` 中打开 `RefExample.usda`，再进入 interpreter。Open `RefExample.usda` in `usdview` and open the interpreter.
  3. 默认遍历：执行 `[x for x in usdviewApi.stage.Traverse()]`，观察 depth-first composed prim 顺序。Use `Stage.Traverse()` to list composed prims in depth-first order.
  4. 按 schema 过滤：用 `if UsdGeom.Sphere(x)` 保留 sphere prim。Filter traversal results with `UsdGeom.Sphere(x)`.
  5. 前后序访问：用 `Usd.PrimRange.PreAndPostVisit(usdviewApi.stage.GetPseudoRoot())` 观察进入/退出事件。Use `PreAndPostVisit` for paired entry and exit visits.
  6. 测试 predicates：在 `usdview` 中 deactivate `refSphere2`，查看 session layer 中的 `active=false`。Deactivate `refSphere2` and inspect the session layer opinion.
  7. 比较遍历视图：用 `Traverse()` 与 `TraverseAll()` 比较 inactive prim 和子树的可见性。Compare `Traverse()` and `TraverseAll()` after deactivation.

关键 API 边界 / Key API Boundaries

  • `Usd.Stage.Traverse()` 是面向常规消费者的便利遍历，默认只访问 active、defined、loaded、concrete prim。`Usd.Stage.Traverse()` applies the default traversal predicates.
  • `Usd.PrimRange.PreAndPostVisit` 适合需要进入/退出子树事件的算法，不只是打印路径。`PreAndPostVisit` is useful when algorithms need enter and exit events.
  • `usdviewApi.stage.GetSessionLayer()` 读取的是 usdview 当前会话的临时 layer，不等于资产文件本身。The session layer stores transient session opinions, not the authored asset layer.
  • `active=false` 会影响 composed scenegraph；inactive prim 的 children 不再作为普通 composed prim 出现。`active=false` changes composed-scenegraph presence for the prim and its children.
  • `TraverseAll()` 不应用默认 predicates，适合诊断被默认遍历跳过的 prim。`TraverseAll()` is diagnostic because it does not apply the default predicates.

Session Layer 与 Deactivate / Session Layer and Deactivate

在 `usdview` 中 deactivate `refSphere2` 后，session layer 会记录临时意见。它适合当前会话调试，不应误认为已经写回资产 layer。

Deactivation in `usdview` authors transient session-layer opinions.

>>> print(usdviewApi.stage.GetSessionLayer().ExportToString())
#usda 1.0

over "refSphere2" (
    active = false
)
{
}
>>> [x for x in usdviewApi.stage.Traverse()]
[Usd.Prim(</refSphere>), Usd.Prim(</refSphere/world>)]

>>> [x for x in usdviewApi.stage.TraverseAll()]
[Usd.Prim(</refSphere>), Usd.Prim(</refSphere/world>),
 Usd.Prim(</refSphere2>)]

常见误读与调试 / Common Misreadings and Debugging

  • 如果遍历结果为空，先确认 `RefExample.usda` 是否在 `usdview` 中打开，且引用的 `HelloWorld.usda` 能被解析。If traversal is empty, verify the opened stage and referenced asset path.
  • 如果路径顺序和预期不同，确认你是在看 depth-first stage traversal，而不是 layer 文件顺序。Traversal order is stage depth-first order, not file text order.
  • 如果 schema 过滤没有匹配，检查是否导入或可访问 `UsdGeom`，以及目标 prim 是否真是 sphere schema。If schema filtering fails, check `UsdGeom` availability and the prim schema.
  • 如果 deactivate 后子 prim 消失，这是 active predicate 的预期结果，不是 reference 丢失。Children disappearing after deactivation is expected predicate behavior, not a lost reference.
  • 如果只想诊断 inactive 或非默认 predicate prim，使用 `TraverseAll()` 或显式 `Usd.PrimRange` 配置，而不是 `Traverse()`。Use `TraverseAll()` or explicit `PrimRange` configuration for diagnostic traversal.
