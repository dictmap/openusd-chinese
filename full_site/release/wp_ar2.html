逐段双语理解 / Paragraph-Level Bilingual Coverage

本页讨论 Ar 资产解析系统从旧接口向 2.0 设计迁移的目标。中文阅读重点是理解 identifier、resolver context、asset info、URI resolver、写入接口和迁移策略之间的关系。它不是简单列出 API 改名，而是在解释 USD 如何把 asset path 从字符串搜索变成更明确、可文档化、可组合、可跨文件系统和 URI 工作流复用的解析协议。Chinese is the main reading path; English identifiers, API names, schema names, tokens, property names, code, commands, and links are retained for verification.

Official section中文主阅读说明 / Chinese-first coverage
Background and Goals 背景说明旧 Ar 接口在 repository、search path、文件系统假设和文档清晰度方面存在限制。Ar 2.0 的目标是让解析过程更明确，便于 resolver 插件实现，也便于调用者理解 identifier 和 resolved asset 的区别。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Tasks 任务列表展示迁移工作不是单点修改，而是一组清理、文档、identifier、repository 移除、resolve 改进、URI resolver 和写入接口设计。读者应把这些任务看成资产解析协议重构。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
General Cleanup 清理任务减少旧接口歧义，便于 resolver 实现者知道哪些方法仍是必要入口，哪些行为应由更明确的 API 承担。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Add Documentation and Examples 文档和示例是 Ar 2.0 的关键，因为 resolver 行为往往跨 studio、文件系统、数据库和网络资产服务。没有示例时，调用者容易把 unresolved path、identifier 和 resolved path 混为一谈。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Add Identifier Concept identifier 概念用于表达 resolver 可稳定理解的资产标识。它不同于用户输入字符串，也不一定等同于本地文件路径；这能支持 package、URI 和上下文相关解析。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Remove Repository and Search Path 移除 repository 和 search path 的旧假设，是为了减少对文件系统目录搜索的依赖。现代资产系统可能来自数据库、云端、package 或自定义 URI，不能只靠传统路径列表。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Improve Resolve and Asset Info Resolve 和 asset info 改进让调用者能同时获得解析结果和相关元数据。调试 asset 找不到时，应检查 resolver context、identifier 创建、asset info 返回和 resolved path，而不只是打印原始字符串。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Rollout and Transition 迁移部分说明旧实现如何过渡到新 API。实际项目中应分阶段适配 resolver 插件、调用端和测试，不要一次性删除所有旧路径逻辑。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.
Proposed API Proposed API 提供具体接口草案。API 名、函数名、类型名和参数必须保留英文；中文只解释职责、调用顺序和兼容风险。This row keeps the official section heading and provides Chinese-first reading guidance for source parity.

中文主阅读路线 / Chinese Main Reading Path

  1. 排查引用失败时，先记录原始 asset path，再看 resolver context 生成的 identifier，最后检查 resolved path 和 asset info，而不是只判断文件是否存在。Operational reading and debugging path.
  2. 实现自定义 resolver 时，应优先定义 identifier 规则和 context 规则，再实现 URI、读取和写入接口，避免把所有逻辑塞进路径字符串解析。Operational reading and debugging path.
  3. 迁移旧工程时，先保留旧测试用例，再按 Ar 2.0 任务逐项替换接口，确保 package、相对路径和上下文解析仍可复现。Operational reading and debugging path.

边界、误读点与调试路径 / Boundaries, Misreads, Debugging

  • Ar 不是普通文件系统包装器，它是 USD 资产解析抽象，必须支持比本地路径更广的资产来源。Boundary or common misread to avoid.
  • identifier、ArResolverContext、Resolve、asset info 等正式术语不能翻译成中文 token。Boundary or common misread to avoid.
  • 本页是提案和迁移设计，具体可用 API 仍需回到当前 OpenUSD API reference 和 resolver 插件实现核对。Boundary or common misread to avoid.
