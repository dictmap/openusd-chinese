范围说明 / Scope Note

本页是 OpenUSD release 教程中“引用 layer”的完整双语参考页。中文主线解释如何把前面教程生成的 `HelloWorld.usda` 作为 asset 引用进新的 stage，如何设置 `defaultPrim`，如何在引用点上 author override，为什么引用不会复制源文件内容，以及如何用 flattened 结果检查 composition。英文页面名、API 名称、路径、文件名、metadata、token、代码和官方链接保留，用于和官方教程核对。

This local bilingual page explains the OpenUSD tutorial for referencing layers in Chinese first while retaining the official title, API names, path literals, file names, metadata names, token literals, code, and links for comparison.

逐段双语理解 / Paragraph-Level Bilingual Coverage

本教程的目标是把之前创建的 `HelloWorld.usda` 引用到一个新的 stage 中。它不再只讨论单个 layer 内的属性读写，而是引入 USD composition 的第一个核心机制：reference composition arc。reference 让一个 prim 可以把另一个 layer 或另一个 prim 的 scene description 组合到当前位置，形成最终 stage 视图。中文阅读时要始终记住：reference 不是文本复制，也不是 import 后静态展开，而是在 composition 阶段把外部意见纳入当前 prim stack。

The tutorial introduces references as composition arcs that bring external scene description into a new stage.

第一步仍然从 `HelloWorld.usda` 开始。脚本用 `Usd.Stage.Open('HelloWorld.usda')` 打开源 stage，再用 `stage.GetPrimAtPath('/hello')` 取得源资产的根 prim。随后调用 `stage.SetDefaultPrim(hello)`，把 layer 的 `defaultPrim` metadata 设置成 `hello`。这个 metadata 是引用工作流的入口提示：如果引用方只指定 asset path 而不指定 prim path，USD 就可以从 `defaultPrim` 找到应该引用的根 prim。

The source layer receives `defaultPrim` metadata so references can target its root prim without spelling a prim path.

`defaultPrim` 的边界很重要。它不是创建新的 prim，也不是把 `/hello` 改名为默认对象；它只是 layer metadata，告诉引用方“这个 layer 的默认根入口是哪个 prim”。如果没有 author `defaultPrim`，引用方仍然可以显式指定目标 prim path，但示例为了简化引用代码，先把源 layer 准备成可直接作为 asset 使用的形式。生产资产发布中，给可引用资产设置稳定的 `defaultPrim` 是常见约定。

`defaultPrim` is layer metadata, not a new prim or a rename operation.

教程还在源 prim 上 author 一个 transform：`UsdGeom.XformCommonAPI(hello).SetTranslate((4, 5, 6))`。这里使用 `UsdGeomXformCommonAPI` 是为了用较简单的一等 API 写常见 xform ops。此时源 `HelloWorld.usda` 中的 `/hello` 带有位移 opinion，后续引用它时，这个 transform 会随源资产一起进入 composed stage。保留 `xformOp:translate` 和 `xformOpOrder` 等名称，是为了让读者能在导出的 `.usda` 中核对实际写入的 xform stack。

The source prim receives a translate opinion through `UsdGeomXformCommonAPI` so the referenced asset carries a transform.

第二步创建新的目标 stage。官方示例用 `Usd.Stage.CreateNew('RefExample.usda')` 创建一个新 layer，并在其中 `DefinePrim('/refSphere')`。这个 prim 起初只是引用点，不包含完整 sphere 内容。随后通过 `refSphere.GetReferences().AddReference('HelloWorld.usda')` 添加 reference arc。因为源 layer 有 `defaultPrim = "hello"`，引用方只需要 asset path 就能把源 `/hello` 的内容组合到 `/refSphere` 位置。

A new stage defines `/refSphere` and adds a reference to `HelloWorld.usda`.

`AddReference('HelloWorld.usda')` 中的字符串由 asset resolver 解析。示例使用相对路径，是因为教程把文件放在同一目录；真实项目里可以是 resolver 支持的 asset path、包内路径或工作室自定义标识符。中文页必须保留 `AddReference` 和 `HelloWorld.usda` 原文，因为调试 reference 失败时，第一步往往就是检查 asset path 是否被 resolver 解析到正确文件。

The reference asset path is resolved by the asset resolver rather than treated as a translated display string.

引用后的导出层看起来很短，这正是 reference 的价值：`RefExample.usda` 只记录 `def "refSphere" ( references = @./HelloWorld.usda@ )` 之类的 arc 信息，而没有复制源文件里的所有 prim 和属性。读者如果只看当前 layer，会觉得内容很少；但打开 composed stage 或 flattened stage 时，`/refSphere` 下会出现源 `/hello/world` 的 sphere 内容。这里必须区分“当前 layer 写了什么”和“stage 组合后看到了什么”。

The referencing layer stores the reference arc, while the composed stage shows the referenced contents.

第三步演示 override。教程取得 `/refSphere` 上的 `UsdGeomXformCommonAPI`，调用 `SetTranslate((0, 0, 0))`，把引用点的 transform 改成新值。导出的 `RefExample.usda` 会出现本地 `xformOp:translate` opinion；它比 referenced layer 中的源 transform 更强，因此 composed 结果里引用点位置被覆盖。这个例子展示了 reference 的非破坏性：你可以在上层 layer 修改组合结果，而不改动 `HelloWorld.usda` 本身。

The tutorial authors an override on the referencing prim that is stronger than the referenced transform.

这一段是 USD composition 强度的入门。reference 带来的源意见位于较弱位置，本地 layer 在同一 prim 上 author 的 opinion 更强。`over "refSphere"` 表示对已经由 reference 引入的 prim 追加或覆盖意见，而不是重新定义整个资产。初学者常把 `over` 理解为删除再创建；更准确地说，它是把额外意见放入 prim stack，让 value resolution 选出最终值。

`over` contributes stronger local opinions to a prim stack without copying or redefining the referenced asset.

第四步引入嵌套 override。源资产中 sphere 位于 `/hello/world`；引用到 `/refSphere` 后，对应路径变成 `/refSphere/world`。教程在新的 layer 中 `OverridePrim('/refSphere/world')`，再使用 `UsdGeom.Sphere(world).CreateDisplayColorAttr()` 或类似 schema API author `primvars:displayColor`。这样做不会修改源 `HelloWorld.usda` 的 sphere，只是在引用实例内部的子 prim 上叠加一个更强颜色 opinion。

The nested override targets `/refSphere/world`, the child prim introduced by the reference.

路径重映射是本页最容易漏掉的概念。源 layer 里的路径是 `/hello/world`，但引用方把源 `defaultPrim` 挂到 `/refSphere`，所以源子 prim `world` 在组合结果中出现在 `/refSphere/world`。如果你在上层错误地 override `/hello/world`，不会影响引用实例。调试引用内部属性时，必须从引用点位置重新推导 composed path，而不是照抄源 layer 的绝对 prim path。

Referenced child paths are mapped under the referencing prim, so overrides must use the composed path.

教程还展示 `stage.Flatten()` 或类似 flattened 输出，用来观察 reference arc 解析后的最终 scene description。flattened layer 会把 reference 引入的内容和上层 override 展开成一个单一结果，便于学习和调试；但 flatten 不是原始工作流的替代品。生产中保留 reference arc 可以复用资产、延迟解析、减少重复，并让源资产更新自动影响引用方；flatten 通常用于诊断、导出或向不支持 composition 的消费者交付。

Flattening is a diagnostic or export view of the composed result, not the same as preserving live reference arcs.

本教程和前两页的关系很清楚：`Hello World` 创建最小 stage，`Inspecting and Authoring Properties` 教你读写 prim 属性，`Referencing Layers` 则把一个已保存 layer 当作可复用资产引入新 stage，并在引用点上 author 更强意见。三者连起来就是 USD authoring 的常见最小闭环：创建资产、检查/修改属性、引用资产、在 shot 或 assembly 层非破坏性覆盖它。

The tutorial connects creation, property authoring, references, and non-destructive overrides into a small USD workflow.

从 API 分组看，本页涉及三类对象。第一类是 `Usd.Stage` 与 `Sdf.Layer` 相关入口，负责打开、创建、保存和 flatten stage；第二类是 `UsdPrim`、`GetReferences()`、`AddReference()`、`OverridePrim()`，负责创建 composition arcs 和本地意见；第三类是 `UsdGeom.XformCommonAPI` 与 `UsdGeom.Sphere`，负责用 schema API author transform 和 displayColor。把三类职责分开，有助于判断 bug 属于文件解析、composition、路径定位，还是几何 schema authoring。

The page uses stage/layer APIs, prim/reference APIs, and geometry schema APIs for different responsibilities.

实际调试 reference 失败时，可以按五步检查：确认 asset path 能被 resolver 找到；确认被引用 layer 有正确 `defaultPrim` 或引用方显式指定了 prim path；确认引用点 prim 路径正确；确认 override 写在 composed path 上；最后用 flattened 输出或 `usdview` 检查最终结果。这个顺序能避免把 asset resolver 问题误判成 override 失效，也能避免把路径重映射错误误判成 reference 没生效。

A practical debugging order is asset path, default prim, reference site, composed override path, and flattened result.

需要保留的关键字面量包括 `HelloWorld.usda`、`RefExample.usda`、`/hello`、`/hello/world`、`/refSphere`、`/refSphere/world`、`defaultPrim`、`references`、`over`、`Usd.Stage.Open`、`Usd.Stage.CreateNew`、`SetDefaultPrim`、`GetReferences`、`AddReference`、`OverridePrim`、`UsdGeom.XformCommonAPI`、`SetTranslate`、`UsdGeom.Sphere`、`CreateDisplayColorAttr`、`stage.GetRootLayer().Save()` 和 `stage.Flatten()`。这些名称是教程可复现和 API 查询的锚点，不应翻译或改写。

The file names, prim paths, metadata names, arc names, API calls, and token literals are the reproducibility anchors for this tutorial.

从中文主阅读路径判断，本页成功的标准不是读者背下 `AddReference` 这一行代码，而是能解释三件事：为什么源 layer 需要 `defaultPrim`，为什么引用方 layer 只写 arc 却能在 composed stage 看到完整 sphere，为什么上层 `over` 和本地 attribute opinion 可以覆盖引用资产而不破坏源文件。能回答这三点，读者就具备继续学习 payload、sublayer、variant、inherits 和 specializes 的 composition 基础。

The main reading path succeeds when the reader understands default prims, composed results, and non-destructive overrides.

还要避免把 reference 和 sublayer 混为一谈。sublayer 通常把另一个 layer 的整层意见合入 layer stack，路径空间保持原样；reference 则把某个 asset 或 prim 的内容挂到当前 prim 的 namespace 位置，因此会产生 `/hello/world` 到 `/refSphere/world` 这样的路径重定位。payload 与 reference 更接近，但 payload 通常用于可延迟加载的大型资产。理解这三个 arc 的差别，可以帮助读者判断什么时候该复用资产实例，什么时候该组织层级意见，什么时候该优化加载成本。

References, sublayers, and payloads solve different composition problems and should not be treated as interchangeable mechanisms.

因此，检查本页是否读懂时，还应确认自己能说明 reference 的 namespace 重定位和本地强意见为何同时成立。

A final reading check is whether the reader can explain namespace remapping together with stronger local opinions.

教程步骤 / Tutorial Steps

  1. 准备源 layer：打开 `HelloWorld.usda`，取得 `/hello`，并把它设置为 `defaultPrim`。Open `HelloWorld.usda`, get `/hello`, and author `defaultPrim`.
  2. 给源 prim 写 transform：用 `UsdGeom.XformCommonAPI(hello).SetTranslate((4, 5, 6))` author 源资产意见。Author a source transform through `UsdGeom.XformCommonAPI`.
  3. 创建引用方 stage：用 `Usd.Stage.CreateNew('RefExample.usda')` 建立新 layer。Create a new referencing stage named `RefExample.usda`.
  4. 建立引用点：定义 `/refSphere`，并用 `GetReferences().AddReference('HelloWorld.usda')` 添加 reference arc。Add a reference from `/refSphere` to `HelloWorld.usda`.
  5. 覆盖引用点 transform：在 `/refSphere` 上写更强的本地 xform opinion。Override the transform on the referencing prim.
  6. 覆盖内部子 prim：在 composed path `/refSphere/world` 上写颜色等属性。Author nested overrides on `/refSphere/world`.
  7. 保存并检查：保存 `RefExample.usda`，用 flattened 输出或 `usdview` 验证 composition 结果。Save and inspect the composed or flattened result.

关键 API 边界 / Key API Boundaries

  • `reference` 是 composition arc；它记录外部 asset 的组合关系，不复制源 layer 的完整文本。A reference is a composition arc, not a textual copy.
  • `defaultPrim` 是 layer metadata；它帮助引用方在未指定 prim path 时定位默认根 prim。`defaultPrim` helps a reference target the layer's default root prim.
  • `over` 表示在已有 prim stack 上追加更强 opinion，不等同于重新定义完整 prim。`over` adds stronger opinions to an existing prim stack.
  • `OverridePrim('/refSphere/world')` 必须使用组合后的路径，而不是源 layer 中的 `/hello/world`。Nested overrides use the composed path under the reference site.
  • `stage.Flatten()` 用于观察最终结果；保留 reference arc 才能保留可复用、非破坏和可更新的 composition 结构。Flattening is useful for inspection, but references preserve reusable composition structure.

代码摘录 / Code Excerpts

from pxr import Usd, UsdGeom

stage = Usd.Stage.Open('HelloWorld.usda')
hello = stage.GetPrimAtPath('/hello')
stage.SetDefaultPrim(hello)
UsdGeom.XformCommonAPI(hello).SetTranslate((4, 5, 6))
stage.GetRootLayer().Save()
stage = Usd.Stage.CreateNew('RefExample.usda')
refSphere = stage.DefinePrim('/refSphere')
refSphere.GetReferences().AddReference('HelloWorld.usda')
stage.GetRootLayer().Save()
UsdGeom.XformCommonAPI(refSphere).SetTranslate((0, 0, 0))
world = stage.OverridePrim('/refSphere/world')
UsdGeom.Sphere(world).CreateDisplayColorAttr([(1, 0, 0)])

print(stage.Flatten())

导出结果检查 / Exported Result Check

引用方 layer 应保留 reference arc，而不是复制源资产；flattened 结果才会显示被引用内容和本地 override 的合成结果。下面只保留关键结构，方便读者核对 `defaultPrim`、`references`、`over` 和 composed path 的关系。

The referencing layer preserves the reference arc, while flattened output shows the composed result.

#usda 1.0
(
    defaultPrim = "hello"
)

def Xform "hello"
{
    double3 xformOp:translate = (4, 5, 6)
    uniform token[] xformOpOrder = ["xformOp:translate"]
    def Sphere "world"
    {
        double radius = 2
    }
}
#usda 1.0

def "refSphere" (
    references = @./HelloWorld.usda@
)
{
    double3 xformOp:translate = (0, 0, 0)
    over "world"
    {
        color3f[] primvars:displayColor = [(1, 0, 0)]
    }
}

常见误读与调试 / Common Misreadings and Debugging

  • 如果引用后看不到内容，先检查 `HelloWorld.usda` 是否能被 asset resolver 找到。If the reference is empty, first verify that the asset path resolves.
  • 如果引用方没有显式 prim path，确认源 layer 是否已经 author `defaultPrim = "hello"`。If no prim path is specified, check the source layer's `defaultPrim`.
  • 如果本地 override 不生效，确认写的是 `/refSphere` 或 `/refSphere/world`，而不是源路径 `/hello` 或 `/hello/world`。If an override does not apply, check the composed target path.
  • 如果 `RefExample.usda` 看起来内容很少，这是正常的；reference layer 只需要记录 arc 和本地强意见。A sparse referencing layer is normal because it stores arcs and local opinions.
  • 如果 flattened 输出符合预期但原 layer 很短，不要把它当成数据丢失；这是 USD composition 的预期分层表达。A short source layer plus rich flattened output is expected in USD composition.
