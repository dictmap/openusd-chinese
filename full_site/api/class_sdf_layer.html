逐段双语理解 / Paragraph-Level Bilingual Coverage

官方描述把 `SdfLayer` 定义为 scene description container。这里的 container 不是运行时 scene graph，也不是 `UsdStage` 的组合视图，而是保存 authored scene description 的底层层文件或匿名层。它可以单独描述一个 component asset，也可以和其他 layers 一起通过 composition 形成更大的 asset、sequence 或 shot 结构。

The official description defines `SdfLayer` as a scene description container. In this context, the container is not a runtime scene graph and not the composed view exposed by `UsdStage`; it is the lower-level layer file or anonymous layer that stores authored scene description. It can describe a component asset by itself, or combine with other layers through composition to form larger asset, sequence, or shot structures.

官方原文还说明 layer 的内容遵循 `SdfData` data model。这意味着 `SdfLayer` 保存的是 specs、fields、metadata、subLayer paths 和 root prim ordering 等结构化描述，而不是直接保存最终求值后的属性值。调试时应先问“这个 opinion 写在哪个 layer”，再问它在 `UsdStage` 上如何被组合和解析。

The official text also states that layer contents follow the `SdfData` data model. This means `SdfLayer` stores structured descriptions such as specs, fields, metadata, subLayer paths, and root prim ordering rather than only final evaluated attribute values. When debugging, first ask which layer authored the opinion, and then ask how it is composed and resolved on a `UsdStage`.

`SdfLayer` 可以是 ephemeral 或 anonymous layer，也可以是通过 `ArAsset` 与 `ArResolver` 访问和序列化的 asset。阅读 `identifier`、`realPath`、resolved path、file format arguments 和 repository path 等接口时，要把“用于定位 layer 的字符串”“resolver 解析出来的路径”和“展示给用户看的名称”分开理解。

An `SdfLayer` may be an ephemeral or anonymous layer, or an asset accessed and serialized through `ArAsset` and `ArResolver`. When reading APIs such as `identifier`, `realPath`, resolved path, file format arguments, and repository path, keep separate the string used to identify a layer, the path resolved by the asset resolver, and the name displayed to users.

`Primary API` 一组通常服务于创建、打开、查找、保存、导出、重载和匿名层管理；`Traversal` 与 `Prims` 一组服务于 layer 内 specs 和 root prims 的枚举；`File I/O` 服务于持久化和字符串导入导出；`Metadata` 与 `Muting` 则分别影响 layer 级元数据和组合可见性。

The `Primary API` group usually covers creation, opening, lookup, saving, export, reload, and anonymous layer management. The `Traversal` and `Prims` groups enumerate specs and root prims inside the layer. The `File I/O` group handles persistence and string import/export. `Metadata` and `Muting` affect layer-level metadata and composition visibility respectively.

常见误读是把 `SdfLayer` 中的 authored opinion 当成 `UsdStage` 上的最终结果。一个 layer 可以写入强弱不同的 opinions；这些 opinions 还要经过 subLayer stack、references、payloads、variants、session layer 和 composition arcs 等规则后，才成为 stage 上看到的 composed value。

A common misunderstanding is to treat an authored opinion inside an `SdfLayer` as the final result on a `UsdStage`. A layer can author opinions with different strengths; those opinions still pass through subLayer stacks, references, payloads, variants, session layers, and composition arcs before becoming the composed value seen on the stage.

与本页高频相邻的类型包括 `SdfPath`、`SdfPrimSpec`、`SdfPropertySpec`、`SdfChildrenView`、`SdfLayerOffset`、`SdfFileFormat`、`ArResolver` 和 `TfToken`。这些类型共同构成了“定位对象、描述对象、遍历对象、保存对象、解析资产路径”的底层 API 语境。

High-frequency neighboring types on this page include `SdfPath`, `SdfPrimSpec`, `SdfPropertySpec`, `SdfChildrenView`, `SdfLayerOffset`, `SdfFileFormat`, `ArResolver`, and `TfToken`. Together they form the lower-level API context for locating objects, describing objects, traversing objects, saving objects, and resolving asset paths.

A scene description container that can combine with other such containers to form simple component assets, and successively larger aggregates. The contents of an SdfLayer adhere to the SdfData data model. A layer can be ephemeral, or be an asset accessed and serialized through the ArAsset and ArResolver interfaces. The SdfLayer class provides a consistent API for accesing and serializing scene description, using any data store provided by Ar plugins.
