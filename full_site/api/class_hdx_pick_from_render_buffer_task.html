逐段双语理解 / Paragraph-Level Bilingual Coverage

`HdxPickFromRenderBufferTask` 是 Hdx 中专门针对 pre-existing ID buffers 运行 picking queries 的 task。官方描述里的重点是“pre-existing”：它不重新绘制整场景，也不重新生成 selection pass，而是复用已经由上游相机和渲染路径生成的 ID buffers，再在其中查询 pick frustum 覆盖的子区域。

`HdxPickFromRenderBufferTask` runs picking queries against pre-existing ID buffers. It reuses existing render-buffer data rather than redrawing the whole scene.

pick frustum 与 camera frustum 的重映射是本页核心。`HdxPickTaskContextParams` 提供交互拾取所需的 pick frustum，但 ID buffers 是用生成它们时的 camera frustum 得到的；任务必须把 pick frustum 映射到该 camera frustum 的坐标关系中，才能正确定位应该读取的 buffer 子区域。

The task remaps the pick frustum from `HdxPickTaskContextParams` into the camera frustum that was used to generate the ID buffers.

`TaskParams` typedef 通常对应 `HdxPickFromRenderBufferTaskParams`。这些参数描述输入 render buffer / ID buffer、frustum、resolution、可能的输出结果和执行选项。它不能与普通 render task 参数混用，因为这里的核心输入不是 scene primitives，而是已经编码过 pick IDs 的 buffer 内容。

`TaskParams` refers to parameters specific to picking from render buffers, not generic render-task parameters.

`Prepare()`、`_Sync()` 和 `Execute()` 要放在 Hydra task 生命周期中理解。`_Sync()` 从 `HdSceneDelegate`、`HdRenderIndex` 或 task params 同步变化；`Prepare()` 准备 query 所需的 GPU / task context 资源；`Execute()` 真正对 ID buffer 子区域运行查询。它们是 `HdEngine::Execute()` 调度链中的阶段，不是普通数据 setter。

`_Sync()`, `Prepare()`, and `Execute()` correspond to synchronization, preparation, and execution inside the Hydra task lifecycle.

该 task 的边界是“从 render buffer 读取 pick 结果”。如果 ID buffer 没有生成、内容不是 pick IDs、生成它的 camera/viewport 与当前 pick context 不一致，或 buffer resolution 与 frustum remap 不匹配，`Execute()` 即使运行也无法产生可信结果。调试时应先验证上游 ID buffer，再验证 remap 和 query。

A valid result depends on valid ID buffers, matching camera/viewport context, correct resolution, and correct frustum remapping.

`IsConverged()` 表示 picking task 自身是否收敛，而不是整个 scene delegate 或所有 upstream render tasks 都已完全同步。在交互式 UI 中，调用方常用它判断是否可以读取稳定 pick result，或是否还要等待上游 render buffer / GPU query 完成。

`IsConverged()` reports convergence of this picking task, not global scene convergence.

`HdSceneDelegate`、`HdRenderIndex`、`HdxTask`、`Hgi` 和 `HdDriver` 的出现说明该类横跨多层边界：scene data 与 task params 提供上下文，render index 提供调度/资源关系，Hgi 和 driver 参与 GPU execution。不要把它只看作 CPU-side hit test，也不要把它当成 USD authoring API。

The class sits across scene-delegate data, render-index/task scheduling, and GPU execution through `Hgi` / `HdDriver`.

调用方的职责是把正确的上下文交给 task：pick request 的区域、对应相机、ID buffer 来源、buffer dimensions 和结果接收位置必须彼此一致。`HdxPickFromRenderBufferTask` 不负责决定场景中哪些 prim 可被选择，也不负责 authoring selection state；这些策略通常在 UI、viewer、selection controller 或更高层的 picking pipeline 中处理。它只把“给定 pick frustum 下，已有 ID buffers 中编码了哪些命中对象”这一底层查询执行出来。

Callers are responsible for providing coherent pick context, camera state, ID-buffer inputs, dimensions, and result storage. The task performs the low-level query; selection policy lives above it.

阅读成员函数时也要区分 public task API 与 protected sync API：`Prepare()` 和 `Execute()` 是执行路径上会被调度的动作，`_Sync()` 则体现 Hydra 对 dirty state、task params 和资源依赖的同步模型。若结果漂移或为空，最常见原因不是类本身“没有拾取能力”，而是前置 ID buffer 与当前 pick frustum 所处的相机空间不同步，或调用方在 buffer 更新前过早读取了 query result。

When reading the member functions, distinguish public execution methods from protected synchronization. Empty or unstable results usually point to stale buffers, mismatched frusta, or premature result reads.

因此，本页的完整双语阅读重点不是罗列所有继承成员，而是说明这个 task 在 Hydra picking pipeline 中的位置：上游负责生成可查询的 ID buffer，中游负责把交互区域映射到正确 frustum，下游再把命中 ID 解释成 viewer 需要的选择结果。

The key reading focus is the task's position in the Hydra picking pipeline: upstream ID-buffer generation, frustum mapping, and downstream interpretation of hit IDs.

常见调试路径是：确认 ID buffers 是否存在且包含 pick IDs；确认生成 ID buffers 的 camera frustum 与当前 pick frustum 是否可正确映射；确认 `_Sync()` 是否拿到了新的 task params；确认 `Prepare()` 是否准备了必要资源；最后检查 `Execute()` 的 query 区域和结果解析。这个顺序可以把 ID buffer 生成问题、frustum remap 问题和 pick-result parsing 问题分开。

Debugging should separate ID-buffer generation, frustum remapping, task-parameter synchronization, query execution, and result parsing.

常见误用包括：把 `HdxPickFromRenderBufferTask` 当成完整 selection renderer；认为 pick frustum 可以直接在 screen coordinates 上读取 buffer；忽略 ID buffer 生成时的 camera frustum；把 `IsConverged()` 当成全场景收敛；或在没有有效 ID buffer 时期待 task 返回对象命中。正确边界是：它复用已有 ID buffers 做局部 picking query，pick frustum、camera frustum、ID buffer、`HdxPickTaskContextParams` 等英文术语必须原样保留。

Common mistakes include treating the task as a full selection renderer, ignoring the camera frustum used to generate the ID buffers, or expecting meaningful results without valid ID-buffer input.

A task for running picking queries against pre-existing id buffers. This task remaps the "pick frustum", provided by HdxPickTaskContextParams , to the camera frustum used to generate the ID buffers.
