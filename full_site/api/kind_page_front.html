The Kind library provides a runtime-extensible taxonomy known as "kinds". Kinds are just TfToken symbols, but the KindRegistry allows for organizing kinds into taxonomies of related/refined concepts, and the KindRegistry::GetBaseKind() and KindRegistry::IsA() queries enable reasoning about the hierarchy and classifying objects by kind. Kinds are useful for classifying scenegraph objects, such as by tagging objects representing the roots of models with the kind of model it represents; it is scenegraph taxonomy that motivates the "builtin" kind hierarchy.
