逐段双语理解 / Paragraph-Level Bilingual Coverage

`TraceEventData` 是 Trace 子系统中随 `TraceEvent` 保存的 payload 值容器。官方说明说它 “holds data that can be stored in TraceEvents”，这里的 data 不是事件本身，也不是 profiler 的时间戳、线程 id、调用栈或事件范围；它只是附加到某个 trace event 上、用于报告和分析的一个值。

`TraceEventData` holds payload values that can be attached to `TraceEvent` records. It does not represent the event timing, thread, stack, or scope itself.

页面列出 6 个 `TraceEventData()` 构造函数重载，反映这个容器可以从 bool、int、uint、float、string 等不同类型构造。构造时选择的类型会决定后续 `TraceEvent::DataType`，也决定调用方应该使用哪个 getter。把它理解成“typed tagged value” 比把它理解成无类型字符串更准确。

The constructor overloads create typed payload values. The selected type determines both `TraceEvent::DataType` and the correct accessor.

`GetType()` 是读取该对象前最重要的分派入口。调用方应先查看 `TraceEvent::DataType`，再选择 `GetBool()`、`GetInt()`、`GetUInt()`、`GetFloat()` 或 `GetString()`。这些 typed getters 不是自动转换接口；在未知或不匹配的类型上读取会造成错误解释，尤其是在把 trace 数据转换成 JSON、UI 表格或统计项时。

Use `GetType()` to dispatch to the matching typed getter. The getters should not be treated as implicit conversion helpers.

`WriteJson()` 把当前 payload 写入 `JsWriter`，因此它位于 trace collection 与 trace report 之间的序列化边界。它不决定事件是否被采集，也不决定 event duration；它只影响该 payload 在 JSON 输出中的类型和值表达。调试 profiling report 字段缺失或类型不对时，应同时检查 payload 构造来源、`GetType()` 分派和 `WriteJson()` 输出。

`WriteJson()` serializes the stored payload through `JsWriter`; it affects report representation, not trace collection or event duration.

`TraceEventData` 与 `TraceEvent::DataType` 的关系类似“值”和“标签”。前者保存具体值，后者描述该值的类型类别。保留 `TraceEventData`、`TraceEvent::DataType`、`GetBool()`、`GetInt()`、`GetUInt()`、`GetFloat()`、`GetString()`、`WriteJson()`、`JsWriter` 等英文标识，可以直接对应源码和官方 Doxygen 页面。

`TraceEventData` stores the value, while `TraceEvent::DataType` labels the value category used for typed access and serialization.

常见误读包括：把 `TraceEventData` 当作完整 `TraceEvent`；把 string 构造当成所有 payload 的统一存储格式；不看 `GetType()` 就调用 getter；把 `WriteJson()` 当成开始或结束 trace 的 API；或者认为它保存线程、时间、scope nesting 等上下文。正确做法是把它放在 Trace event 的 payload 层阅读，再到 `TraceEvent` 或 Trace collector 相关页面查看事件上下文。

Common mistakes come from treating this payload container as the whole event or using accessors without checking the data type first.
