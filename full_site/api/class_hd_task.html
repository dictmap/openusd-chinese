逐段双语理解 / Paragraph-Level Bilingual Coverage

`HdTask` 是 Hydra 在一次 render 执行中调度的抽象工作单元。官方描述说它可以用于准备 resources、运行 3D render passes、运行 compositing / color correction 这类 2D render passes，或协调应用程序与其他 renderers 的集成。重点是：task 是 `HdEngine::Execute()` 调度的执行节点，不是 USD prim、不是 schema，也不是 `HdSceneDelegate` 中的数据对象。

`HdTask` represents a unit of work during a Hydra render. It is scheduled by `HdEngine::Execute()` and is distinct from USD prims, scene-delegate data objects, and schema wrappers.

`HdTask` 的生命周期通常围绕 `Sync()`、`Prepare()` 和 `Execute()` 理解。`Sync()` 根据 dirty bits、`HdSceneDelegate`、`HdRenderIndex` 和 task id 同步任务参数；`Prepare()` 在真正执行前准备资源、填充 task context 或整理依赖；`Execute()` 执行派生任务的实际逻辑，例如发起 render pass、做后处理、读取 AOV、绘制辅助可视化或把结果交给应用层。

The main lifecycle is `Sync()`, `Prepare()`, and `Execute()`: synchronize task state, prepare resources or context, then perform the concrete work implemented by the subclass.

`Sync()` 不等同于执行渲染。它的任务是把场景委托中的 task parameters、dirty state 和相关依赖更新到 task 对象中。`GetInitialDirtyBitsMask()` 决定初始同步需要关注哪些 dirty bits；如果 mask 过宽，会造成无意义同步成本；如果过窄，task 可能错过参数变化，表现为 viewport 不刷新、pick 结果过期或 render pass 使用旧状态。

`Sync()` updates task state rather than rendering by itself. `GetInitialDirtyBitsMask()` controls which dirty bits are considered on initial synchronization.

`Prepare()` 是执行前的资源和上下文整理阶段。许多 Hdx task 会在这里读取或写入 task context data，建立 render pass 需要的输入，或把一个 task 生成的数据交给后续 task。它常常不直接绘制，但如果漏掉必要准备，`Execute()` 阶段可能拿不到 AOV、selection buffer、camera state、lighting state 或 renderer-specific driver resource。

`Prepare()` is the pre-execution setup phase. It is where tasks can arrange resources, populate task context data, and prepare inputs for later execution.

`Execute()` 是实际工作入口。派生类可以在这里调用 `HdRenderPass`，执行 color correction、compositing、selection、picking、shadow、present 或 debug visualization 等任务。不要把 `HdTask` 理解为固定的 draw call 包装器：它也可以是资源准备、数据转换、渲染后处理或应用集成步骤，具体含义由 subclass 决定。

`Execute()` performs the concrete task behavior, which may be rendering, post-processing, picking, selection, presentation, debugging, or integration work depending on the subclass.

task context 是 `HdTask` 协作的关键机制。`_GetTaskContextData()`、`_HasTaskContextData()` 和 `_GetTaskParams()` 帮助派生任务读取共享数据或自身参数。多个 task 在同一 `HdEngine` 执行序列中可以通过 context 传递中间结果；因此调试 task 链时要同时看执行顺序、context key、value 类型、缺省值和数据所有权，而不是只看单个 task 的 `Execute()`。

Task context data allows tasks in an engine execution sequence to share parameters and intermediate results. Debugging task chains requires checking context keys, value types, execution order, and ownership.

`GetRenderTags()` 用于表达 task 关心的 render tags。它可以影响哪些对象进入某个 task 的处理范围，但不应与 USD purpose、collection、render visibility 或 Hydra collection predicate 混为一谈。它是 task/render pipeline 层的过滤信息，常与 render pass、collection、selection 或 debug visualization 组合使用。

`GetRenderTags()` describes render-tag filtering at the task/render pipeline level. It is related to filtering, but it is not the same API layer as USD purpose, collections, or visibility metadata.

`IsConverged()` 表示任务级结果是否稳定。progressive rendering、path tracing、异步 AOV 生成、selection/picking readback 或后台 GPU work 都可能导致 task 在多帧之间继续变化。`IsConverged()` 可以帮助 viewport 或 render loop 决定是否继续调度，但它不是最终图像质量证明，也不保证所有子系统都已经没有 pending work。

`IsConverged()` reports task-level stability for scheduling decisions. It is not a universal proof of image quality or of global renderer idleness.

`_GetDriver()` 暴露 task 访问 driver / renderer integration 数据的入口。driver 通常来自应用或渲染后端集成层，可能携带 GPU context、render delegate resource、interop handle 或应用私有状态。派生 task 应把它当成集成边界使用，不要把 driver 与 scene delegate、render index 或 USD stage 混为同一层。

`_GetDriver()` is an integration boundary for renderer or application driver data. It should not be confused with the scene delegate, render index, or USD stage.

派生类列表说明 `HdTask` 是一个广义执行框架：`HdxRenderTask` 负责常规渲染，`HdxPickTask` 与 `HdxPickFromRenderBufferTask` 处理 picking，`HdxSelectionTask` / `HdxColorizeSelectionTask` 处理选择可视化，`HdxColorCorrectionTask`、`HdxPresentTask`、`HdxVisualizeAovTask` 处理后处理和显示，`HdxShadowTask`、`HdxOitRenderTask` 等处理特定渲染技术。阅读本页时应把 `HdTask` 看成这些任务的共同协议。

Derived task classes show that `HdTask` is a broad execution protocol for rendering, picking, selection visualization, post-processing, presentation, shadows, OIT, and other Hydra workflows.

常见误用包括：在 `Sync()` 中执行昂贵渲染工作；忽略 dirty bits 导致 task 参数不更新；在 `Prepare()` 和 `Execute()` 之间错误假设 context data 已存在；把 `IsConverged()` 当成最终质量标准；把 render tags 当成 USD purpose；把 task 当成 scene prim 或 renderer backend 对象；以及在派生 task 中绕过 `HdEngine` 调度顺序直接访问未准备好的资源。

Common mistakes include rendering during `Sync()`, ignoring dirty bits, assuming context data exists before `Prepare()`, treating `IsConverged()` as final quality, confusing render tags with USD purpose, and bypassing `HdEngine` scheduling assumptions.
