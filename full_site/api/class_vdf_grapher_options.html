逐段双语理解 / Paragraph-Level Bilingual Coverage

`VdfGrapherOptions` 是 `VdfGrapher` 的输出配置对象。官方说明说它 “can be used to configure aspects of VdfGrapher's output”，这里的 output 指 graph visualization / graph dump 的呈现结果，而不是 Vdf computation 的数值结果。阅读本页时应先把三层职责分开：`VdfNode` 描述图中的计算节点，`VdfContext` 在 callback 执行时提供输入输出访问，`VdfGrapherOptions` 只决定这些图结构被怎样筛选、着色、标注和排版。

`VdfGrapherOptions` configures the presentation emitted by `VdfGrapher`; it is not an evaluator and does not change graph execution or computed values.

页面中的 `NodeFilterCallback`、`NodeStyleCallback` 和 `NodeLimitVector` 是读懂该类的核心类型。`NodeFilterCallback` 负责决定某个 `VdfNode` 是否出现在输出图里；`NodeStyleCallback` 负责为节点提供颜色、注释或 display style；`NodeLimitVector` / `NodeLimit` 用来约束图展开范围。它们共同服务于“把大型 Vdf network 缩小成可读诊断图”的目标，而不是修改 network 的拓扑或依赖关系。

The callback and node-limit types are filtering and styling hooks for generated graph output; they do not mutate the underlying `VdfNetwork` topology.

`DisplayStyle` 是输出图的视觉策略。它与 `GetColor()`、`GetAnnotation()`、`GetDisplayStyle()` 这组查询一起决定节点以什么颜色、说明文字和样式出现在 graph dump 中。调试时如果图中节点样式不符合预期，首先检查这些 option 或 callback，而不是把它解释成 Vdf evaluation state 改变。

`DisplayStyle`, color, and annotation settings describe how graph elements are rendered in the dump, not how values are evaluated.

`AddNodeToGraph()` 与 `GetNodesToGraph()` 控制显式纳入图输出的节点集合。`DebugNameFilter()` 允许按 debug name 收窄结果，适合在大型网络中只观察某个计算链、某类 node 或某个命名片段。它们的语义是 visualization scope selection；如果图里没有出现某个节点，可能是 filter/limit/name 规则排除了它，不应立刻推断该节点不存在或没有参与计算。

Node selection APIs define the visualization scope. Missing nodes in a dump can be a filtering artifact rather than proof that the node is absent from execution.

`GetDrawMasks()` 与 `GetDrawAffectsMasks()` 控制是否绘制 dependency mask / affects mask 信息；`GetDrawColorizedConnectionsOnly()` 进一步压缩输出，让图只突出带颜色语义的连接。mask 是 Vdf 局部求值、请求传播和依赖裁剪的重要诊断线索，但在这里它们只是被可视化；真实的 mask 计算和 evaluation scheduling 仍属于 Vdf 执行系统。

Mask drawing options expose dependency information in the graph output, while the actual mask computation remains part of Vdf evaluation.

`GetPageWidth()`、`GetPageHeight()`、`GetPrintSingleOutputs()` 和 `GetOmitUnconnectedSpecs()` 面向导出版面和噪声控制。页面尺寸决定生成图的排版边界；single output 打印策略决定是否把只有一个 output 的节点展开得更详细；省略 unconnected specs 可以减少孤立输入输出造成的视觉噪声。这些选项通常用于让调试报告更可读，而不是追求完整列出所有内部对象。

Page-size and noise-control options trade completeness for readability in exported diagnostic graphs.

本页和 `VdfGrapher`、`VdfNode`、`VdfConnectionVector`、`VdfObjectPtr`、`TfToken` 的关系也需要明确：`VdfGrapher` 执行图导出，`VdfGrapherOptions` 只承载导出参数；`VdfNode` 是被观察对象；`VdfConnectionVector` 等类型描述连接集合；`TfToken` 常出现在命名、debug name 或 tokenized option 中。不要把 options 对象当作 graph owner，也不要把生成的图片当作 profiling trace。

The options object is adjacent to grapher, node, connection, and token types, but it owns neither the graph nor the runtime profiling evidence.

常见误读包括：把 graph dump 中隐藏的节点当成运行时未执行；把 color / annotation 当成求值结果；把 mask 绘制开关当成 mask 计算开关；把 `NodeFilterCallback` 当成调度谓词；把 page width / height 当成图结构限制。正确用法是：先用 filter 和 limit 缩小范围，再用 style/mask/annotation 增强可读性，最后把导出的图与 evaluation log、profiling 或节点 debug name 交叉核对。

Common mistakes come from treating visualization options as execution controls. Use them to inspect and explain graph structure, then corroborate with evaluation logs or profiling data.
