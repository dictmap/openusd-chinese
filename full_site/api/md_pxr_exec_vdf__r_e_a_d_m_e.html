中文主阅读路径 / Chinese Main Reading Path

本页是 Vdf 模块的 README 型入口页。官方正文非常短，但它定义了 OpenExec 栈里最底层的一个核心概念：vdf library 提供 building and evaluating vectorized dataflow netorks 的 foundation。这里的 source snapshot 保留了官方拼写 netorks，本页也保留该字面词用于 parity，但中文解释按“networks / 网络”理解。The vdf library provides the foundation for building and evaluating vectorized dataflow netorks.

阅读本页时不要把 Vdf 当作普通容器库，也不要把它直接等同于 ExecUsd。ExecUsd 负责把 USD scene 和 schema computation 接到 OpenExec；Vdf 关心的是 runtime 侧如何用 dataflow network 表达 computation 依赖、数据从 node outputs 流到 node inputs、以及后续 evaluation 如何沿网络执行。Vdf is the vectorized data-flow foundation beneath higher OpenExec layers.

官方唯一 section 是 Key Concepts。其中 VdfNetwork embody 一个 dataflow network；网络由执行 computation 的 VdfNode 组成；数据从节点输出流向节点输入；连接关系由 VdfConnection 表达。中文主线要围绕这四个名词展开，而不是扩写成无关 USD authoring 或渲染教程。Key Concepts define VdfNetwork, VdfNode, outputs, inputs, and VdfConnection.

Key Concepts 逐项解释 / Key Concepts Explained

  • VdfNetwork。它是 dataflow network 的整体容器，描述哪些 nodes 存在、它们之间如何连接、数据怎样沿边流动。对 OpenExec 读者来说，VdfNetwork 是 System Design 中 execution network / vectorized data flow network 的底层表达之一。VdfNetwork embodies a dataflow network.
  • VdfNode。node 是执行 computation 的单位。它不是 USD prim，也不是 schema class；它是 runtime network 中可以被调度、求值和连接的计算节点。把 VdfNode 和 UsdPrim 混淆，会导致调试时把 scene authoring 问题与 execution graph 问题混在一起。VdfNode performs computation inside the network.
  • node outputs 与 node inputs。官方强调 data flows from node outputs to node inputs。这个方向是理解 dependency 和 evaluation 的关键：一个节点的输出可以成为另一个节点的输入，输入变化会影响依赖它的 downstream computation。Data flows from node outputs to node inputs.
  • VdfConnection。连接对象表达 output 到 input 的关系。它不是用户点击导航链接，也不是 USD relationship；它是 dataflow network 内部 dependency edge。调试 network 缺边、错边或 stale value 时，连接语义通常是第一批要检查的对象。VdfConnection represents data-flow connections.

栈中位置 / Stack Position

Vdf 位于 OpenExec 系统更底层的位置。上层 ExecUsd 关心 USD stage 如何进入系统，Exec 关心 execution system core，Esf/EsfUsd 关心 execution scene foundation 和 USD 适配，而 Vdf 关心用向量化 data flow graph 表达并评估计算。这个分层能避免读者把“找不到 schema computation”直接归咎于 Vdf；那类问题通常先查 plugin registration、ExecUsd ingestion 或 schema mapping。Vdf sits beneath ExecUsd, Exec, and EsfUsd as the data-flow foundation.

从系统设计角度看，Vdf 的关键词与前一页 OpenExec System Design 互相呼应。System Design 讨论 Compilation、Scheduling、Evaluation、Network、Data Managers 和 Executors；本页则把 Network 层的基础词汇具体化为 VdfNetwork、VdfNode 和 VdfConnection。如果读者还没有理解三阶段执行模型，应先回到 System Design。Vdf gives concrete vocabulary for the network layer discussed in System Design.

从 API 选择角度看，本页不是 class reference 的替代品。它告诉你 Vdf 模块解决什么问题；具体成员函数、数据结构和调度细节仍要沿 Doxygen class/source 页继续阅读。中文页不能伪造未在短源页出现的成员表，也不能把 Vdf 扩写成完整 OpenExec 用户教程。This README page explains the module role, not every class member.

逐段双语理解 / Paragraph-Level Bilingual Coverage

标题。Vdf: Vectorized Data Flow 是模块名和概念名。翻译时保留 Vdf、Vectorized Data Flow 原文，避免把它弱化成普通“数据流”。它在 OpenExec 语境中指可以被编译、调度、缓存和并行评估的运行时数据流网络基础。The title names the vectorized data-flow foundation.

开场句。官方说 vdf library provides the foundation for building and evaluating vectorized dataflow netorks。这里的重点有两个：一是 building，说明 Vdf 参与构建网络结构；二是 evaluating，说明它不仅描述静态图，也服务运行时求值。源页拼写为 netorks，中文语义按 networks 解释。The vdf library provides the foundation for building and evaluating vectorized dataflow netorks.

Key Concepts 标题。这说明页面不是 changelog，也不是教程步骤，而是在定义读者后续阅读 Vdf class/source 页需要先掌握的词汇。若不理解 VdfNetwork、VdfNode 和 VdfConnection，继续读 executor、scheduler 或 invalidation 相关页面时很容易把节点、连接、输入、输出混淆。Key Concepts introduces the vocabulary needed for deeper API pages.

概念句。VdfNetwork embodies a dataflow network consisting of nodes (VdfNode) that perform computation with data flowing from node outputs to node inputs (see VdfConnection)。这句话是本页的完整核心：network 是整体，nodes 负责 computation，outputs/inputs 给出方向，connection 给出依赖边。VdfNetwork, VdfNode, node outputs, node inputs, and VdfConnection define the core model.

边界、误读点与调试路径 / Boundaries and Debugging

  • 如果问题是 schema computation 完全没有注册，优先看 ExecUsd、Tutorial 2 和 plugin metadata，不要先查 VdfConnection。Registration failures usually belong above Vdf.
  • 如果问题是网络中某个 computation node 缺失或连接方向错误，才进入 Vdf 语义：检查 VdfNetwork 是否包含目标 VdfNode，以及 output 到 input 的 VdfConnection 是否正确。Network shape and connection errors belong to Vdf concepts.
  • 如果 computed value 陈旧，先判断是 input 没有通过 connection 表达、invalidation 没有传播，还是 cache/evaluation 层没有重新运行目标节点。Stale values can come from missing dependency edges or invalidation gaps.
  • 如果性能差，区分网络构建成本、schedule 变化、以及节点 computation 本身成本。Vdf 主要帮助理解 dataflow network 的结构和 evaluation 基础，不直接替代上层 profiling 工具。Performance diagnosis should separate network construction, scheduling, and node computation cost.
  • 如果阅读路径不清楚，先从 OpenExec Overview、Tutorial 1/2、System Design 到 ExecUsd README，再进入本页理解底层 Vdf 词汇。The local click path reaches Vdf after OpenExec concepts and ExecUsd.

网络模型验收清单 / Network Model Checklist

为了把这个短页读实，建议用一条最小 mental model 检查自己是否理解：一个 VdfNetwork 里有多个 VdfNode；每个节点可以产生 outputs，也可以消费 inputs；VdfConnection 把上游 output 接到下游 input；evaluation 时，系统沿这些依赖关系决定哪些 computation 需要运行，哪些结果可以复用，哪些变化需要传播。这个模型正是 OpenExec 能从“场景里有很多可计算值”走向“可调度、可缓存、可并行求值”的基础。A VdfNetwork connects computing nodes through output-to-input dependencies.

排查时也可以按这条模型逐步缩小范围。先问网络是否存在，确认目标 computation 是否已经变成 VdfNode；再问连接是否存在，确认 upstream output 是否真的接到 downstream input；然后问数据方向是否正确，避免把 input/output 反向理解；最后问 evaluation 是否看到这些依赖，避免 stale value 或缺失 invalidation。这样比直接搜索所有 class reference 更稳定，也更符合官方短页的概念意图。Debugging follows network presence, node presence, connection presence, and evaluation visibility.

如果读者正在实现或审查一个 OpenExec 功能，可以用本页做分层检查：业务语义留在 schema 和 ExecUsd，执行计划留给上层 scheduler/executor，Vdf 关注的是数据流网络中节点和连接是否把依赖表达清楚。只要这个边界清楚，后续阅读具体 Vdf 类时就能知道每个 API 是在服务网络结构、连接关系还是求值过程，也能更快定位错误属于哪一层，并避免盲目修改无关模块和误判根因。Vdf is the layer where dependency structure becomes explicit network form.
