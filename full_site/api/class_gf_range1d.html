逐段双语理解 / Paragraph-Level Bilingual Coverage

官方定义把 `GfRange1d` 称为 “Basic type: 1-dimensional floating point range”。这里的 range 也可理解为 interval：它用一对端点 `min` 与 `max` 描述一维 double 精度闭区间。它属于 `Gf` Basic Geometry 的基础数学值类型，不是 USD prim，也不会自己表示 layer、attribute 或 authored extent。

The official definition describes `GfRange1d` as a one-dimensional floating point range, or interval, represented by minimum and maximum endpoints.

本页最容易误读的是 empty range。官方明确说 “An empty range is one where max < min”，默认空值是 `[FLT_MAX,-FLT_MAX]`。这个状态看起来像反转的非法端点，但在 API 语义里它是有意设计的空区间，适合从默认构造开始逐步 `ExtendBy()` 数据点或子区间，最终形成有效 range。

An empty range is defined by `max < min`, and the default empty range uses the reversed endpoint convention `[FLT_MAX,-FLT_MAX]`.

`MinMaxType` 与 `ScalarType` 说明本类的端点存储和标量精度。`ScalarType` 对应 double 精度；相关链接 `GfRange1f` 是 float 精度版本。切换精度时要注意端点比较、序列化误差和 bounds 累积误差，尤其是在从大量采样值、time samples 或几何 extent 中构造一维范围时。

`ScalarType` identifies the scalar precision for the range; `GfRange1f` is the related float precision range type.

3 个 `GfRange1d()` 构造函数覆盖默认空区间、由端点构造区间，以及从相关类型转换或复制的场景。构造时调用方要明确端点顺序：如果传入 `min > max`，它不一定是“自动排序失败”，而可能按照 empty range 规则被视为空区间。需要自动整理端点的上层代码应在调用前自己处理。

The constructor overloads cover the default empty state, explicit endpoint construction, and related range construction or copying; endpoint order has semantic meaning.

`GetMin()` 与 `GetMax()` 只是读取当前端点，不会验证区间是否为空，也不会自动交换端点。`GetMidpoint()` 与 `GetSize()` 对普通闭区间很直接，但对 empty range 调用时应先检查调用方语义：空区间的 midpoint 或 size 不是几何中心，而是当前端点状态推导出的数学结果。

`GetMin()` and `GetMax()` return endpoints as stored; callers should check empty-range state before interpreting midpoint or size values.

`Contains()` 有两个重载，分别用于判断一个点或另一个 range 是否包含在当前区间内。因为本类遵循 interval mathematics，端点相等通常表示闭区间边界上的包含，而不是“外部点”。如果当前对象是 empty range，包含测试的结果可能和直觉不一致，所以在使用 `Contains()` 前最好先明确 `IsEmpty()` 或等价状态。

`Contains()` tests containment of a value or another range according to interval mathematics, including endpoint boundary semantics.

`ExtendBy()` 是累积 bounds 的核心方法。它会扩展当前 `min` / `max` 以包含一个点或另一个 range；从默认 empty range 开始连续调用 `ExtendBy()` 是常见模式。调试 bounds 过大或过小时，应检查是否漏掉了某个输入点、是否把 empty range 当作有效初值、以及是否在错误坐标空间中扩展了区间。

`ExtendBy()` grows the range to include a point or another range, making the default empty range useful as an accumulation starting state.

`GetUnion()` 返回两个区间的并集视图；`GetIntersection()` 和 `Intersection()` 处理交集。交集可能为空，这时会回到 `max < min` 的 empty range 规则。端点刚好相等时，交集不是空，而是长度为 0 的单点闭区间，这一点在碰撞、裁剪、时间范围和参数范围计算中很重要。

Union and intersection methods follow interval mathematics; equal endpoints can represent a valid zero-size closed range, while `max < min` represents empty.

`GetDistanceSquared()` 给出点到区间的平方距离。点在区间内部或端点上时距离为 0；点在区间外时返回到最近端点的平方距离。使用 squared distance 可以避免不必要的 `sqrt`，适合排序、阈值比较或粗略剔除，但如果需要实际距离，调用方必须自己开方。

`GetDistanceSquared()` returns squared distance from a value to the interval, avoiding a square root when only comparisons are needed.

“All operations are component-wise” 在一维类型里看似多余，但它让 `GfRange1d` 与 `GfRange2*`、`GfRange3*` 的 mental model 保持一致：多维 range 的每个维度都按端点区间独立处理。理解这一点有助于把一维参数范围、时间范围、UV 范围和三维 bounding box 的 extent 代码统一起来。

The component-wise note aligns the one-dimensional type with higher-dimensional Gf range types used for bounds and extents.

本页链接到 `Deprecated:`，说明部分旧接口或旧命名可能仍出现在 Doxygen 中。维护新代码时应优先使用当前非 deprecated 的 `GetUnion()`、`GetIntersection()`、`ExtendBy()` 等接口，并在迁移旧代码时确认旧接口的 empty-range 行为、端点闭合规则和返回值是否与新写法完全一致。

Deprecated entries should be treated as migration guidance; new code should prefer current range APIs and verify empty-range behavior when porting old calls.

实际调试路径可以固定为：先检查区间是否 empty，再检查 `min` / `max` 是否来自同一坐标或参数空间；然后确认 `ExtendBy()` 是否覆盖所有输入样本；接着区分空交集和零长度交集；最后检查 double/float 转换是否改变了边界。这样能避免把 range 计算错误误判成 USD composition、renderer culling 或几何拓扑问题。

A practical debugging path is: check empty state, endpoint coordinate space, accumulated samples, empty versus zero-size intersections, and precision conversion before blaming composition or rendering behavior.

常见用例包括时间区间、参数域、标量 bounds、动画采样范围和一维投影范围。`GfRange1d` 只关心数值端点，不知道这些端点代表 timeCode、distance、UV 参数还是权重；因此上层代码必须保证单位一致、坐标空间一致、端点含义一致。混用 frame number 与 seconds、局部坐标与世界坐标，都会让 interval API 看起来正确但业务结果错误。

Common uses include time intervals, parameter domains, scalar bounds, animation sample ranges, and one-dimensional projected extents; the range stores numbers but not their semantic units.

如果从数据流中累积范围，推荐先创建默认 empty range，再对每个有效样本调用 `ExtendBy()`。不要用 `[0,0]` 作为通用初始值，除非 0 确实属于业务数据；否则所有正值或所有负值样本都会被人为拉向 0。这个初始化错误在 bounds、time samples 和 normalized parameter 处理中很隐蔽。

For accumulation, the default empty range is safer than `[0,0]` unless zero is a real sample in the domain.

诊断结论：先查 empty，再查端点单位。

Diagnostic summary: check empty state, then endpoint units.
