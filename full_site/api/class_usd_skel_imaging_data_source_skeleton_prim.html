еңЁе®һйҷ…жҺ’жҹҘдёӯпјҢ`GetNames()` дёҺ locator еӨұж•ҲжҠҘе‘ҠиҰҒдёҖиө·зңӢпјҡеүҚиҖ…иҜҙжҳҺеҪ“еүҚ data source е…¬ејҖдәҶе“Әдәӣеӯ—ж®өпјҢеҗҺиҖ…иҜҙжҳҺе“Әдәӣеӯ—ж®өеӣ дёә USD еҸҳеҢ–иҖҢеә”иў«йҮҚж–°жӢүеҸ–гҖӮдёӨиҖ…дёҚдёҖиҮҙж—¶пјҢеёёи§ҒеҺҹеӣ жҳҜ adapter жІЎжңүжҠҠ skeleton-specific еӯ—ж®өеҠ е…Ҙ containerпјҢжҲ–иҖ… dirty locator жІЎжңүзІҫзЎ®жҳ е°„еҲ° skeleton data source еӯҗи·Ҝеҫ„гҖӮ

GetNames() and data source locators should be checked together when diagnosing missing or stale skeleton data.

Inherited data-source classes separate generic prim data from skeleton-specific data.

еёёи§ҒиҜҜиҜ»еҢ…жӢ¬пјҡжҠҠ `GetNames()` еҪ“жҲҗ USD еұһжҖ§еҲ—иЎЁпјӣжҠҠ `Get()` еҪ“жҲҗзӣҙжҺҘиҜ»еҸ– `UsdAttribute`пјӣжҠҠ `UsdSkelImagingDataSourceSkeletonPrim` еҪ“жҲҗ `UsdSkelSkeleton` зҡ„ authoring APIпјӣжҠҠ data source еұӮй”ҷиҜҜеҪ“жҲҗеҠЁз”»жұӮи§Јй”ҷиҜҜпјӣжҲ–иҖ…еҝҪз•Ҙ `UsdImagingDataSourceStageGlobals` дёӯзҡ„жҲҗеғҸдёҠдёӢж–ҮгҖӮжӯЈзЎ®жҺ’жҹҘйЎәеәҸйҖҡеёёжҳҜпјҡзЎ®и®Ө prim жҳҜжңүж•Ҳ SkeletonпјҢзЎ®и®Ө `SdfPath` е’Ң stage globals жӯЈзЎ®пјҢжЈҖжҹҘ `GetNames()` жҡҙйңІзҡ„ tokenпјҢжҢү token и°ғ `Get()`пјҢеҶҚжІҝ locator / adapter / scene index иҝҪиёӘж•°жҚ®жҳҜеҗҰиў« Hydra ж¶Ҳиҙ№гҖӮ

Common debugging separates USD attribute authoring, data-source exposure, locator invalidation, and downstream animation or rendering behavior.
