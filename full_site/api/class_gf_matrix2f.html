逐段双语理解 / Paragraph-Level Bilingual Coverage

`GfMatrix2f` 是 Gf Linear Algebra 模块中的 2x2 `float` matrix 值类型。它适合表达二维线性变换、二维坐标基变化、小型矩阵运算和局部数值工具逻辑；它不是 USD scene graph 对象，也不是 `UsdGeomXformOp`、prim、attribute 或 authored metadata。

`GfMatrix2f` is a 2x2 `float` matrix value type in Gf Linear Algebra. It is a low-level numeric type, not an authored USD scene object or transform op.

官方摘录说本类存储 2x2 `float` 元素，并按 row-major order 定义，因此 `matrix[i][j]` 表示第 `i` 行第 `j` 列。这个细节不是实现噪声，而是阅读构造函数、`operator[]`、`GetRow()`、`GetColumn()`、调试打印和外部库互操作时的基础约定。

The reference defines the matrix as row-major, with `matrix[i][j]` indexing row `i`, column `j`. This affects construction, indexing, row/column access, debugging, and interop.

`ScalarType` 对本类是 `float`。这让 `GfMatrix2f` 适合轻量二维数学和渲染/工具侧数据流，但它的精度低于 `GfMatrix2d`。如果矩阵来自 double precision 计算、需要稳定求逆、或会经过长链路反复相乘，应明确判断是否该使用 `GfMatrix2d`，不要把 `GfMatrix2f` 默认当成高精度容器。

For this class `ScalarType` is `float`. Use `GfMatrix2d` when double precision or more stable numerical behavior is required.

二维 2x2 矩阵表达的是线性部分：旋转、缩放、剪切、坐标基变化或一般 linear map。它通常不携带平移项；如果要表达完整 2D affine transform，需要额外的 translation vector 或更高维齐次矩阵。把 `GfMatrix2f` 当作完整 transform container 是常见误读。

A 2x2 matrix represents the linear part of a 2D transform. It does not by itself store translation; a full 2D affine transform needs additional data or a homogeneous matrix.

构造和设置 API 用来建立矩阵状态，例如默认构造、从标量或数组构造、从行/列数据构造，以及 `SetIdentity()`、`SetZero()` 等基础赋值。阅读这些方法时要区分“设置当前矩阵”为某个状态和“在已有矩阵上继续组合”；如果要组合，应显式使用矩阵乘法并核对行向量/列向量约定。

Construction and setup APIs create or assign a matrix state. They should not be read as implicit append or composition operations.

`GetDeterminant()` 是判断二维矩阵可逆性、面积缩放和退化情况的关键。determinant 接近 0 时，矩阵可能是 singular 或 near-singular；此时 `GetInverse()` 的结果会对 `float` 误差非常敏感。实际调试中，应把 determinant、容差和业务允许误差一起看，而不是只看逆矩阵函数是否返回了一个对象。

`GetDeterminant()` is central for invertibility and degeneracy checks. `GetInverse()` on singular or near-singular matrices is sensitive to `float` precision and tolerance choices.

`GetTranspose()`、`GetRow()`、`GetColumn()`、`data()`、`Get()` 和 `operator[]` 都围绕矩阵元素访问展开。它们看似只是便利函数，但在排查“结果像是被转置”“行列顺序反了”“外部库读写不一致”这类问题时非常重要；中文层必须保留这些 API 名称和 `matrix[i][j]` 符号。

Element-access APIs such as `GetTranspose()`, `GetRow()`, `GetColumn()`, `data()`, `Get()`, and `operator[]` are essential when diagnosing transpose or row/column-order mistakes.

矩阵乘法不是可交换运算，二维矩阵组合也一样。与 `GfVec2f` 搭配时，要确认代码采用 row vector 还是 column vector 思路；外部数学库、DCC 工具、shader 或 UI 坐标系统可能有不同约定。最安全的做法是明确输入向量在左侧还是右侧、矩阵是否需要 transpose、坐标系单位和 handedness 是否一致。

Matrix multiplication is not commutative. When combining `GfMatrix2f` with `GfVec2f` or external math libraries, confirm vector convention, transpose needs, units, and coordinate-system assumptions.

相关类型说明了本类边界：`GfMatrix2d` 是 double precision 对应类型，`GfVec2f` 是二维 float vector，`GfMatrix4f` 则面向更常见的 3D homogeneous transform。把 2x2 矩阵扩展到 3D 或 affine 语义时，应明确哪些信息会丢失，例如 translation、projective 分量、z 轴信息或 double precision。

Related types include `GfMatrix2d`, `GfVec2f`, and larger matrix types such as `GfMatrix4f`. Moving between them can drop translation, projective terms, z-axis data, or precision.

常见误用包括：把 `GfMatrix2f` 当作 USD 可编辑属性或 prim；忘记它只存 2x2 线性部分；在需要平移时仍使用 2x2 矩阵；不检查 determinant 就求 inverse；把 column-major 数据直接按 row-major 读取；以及忽略 `float` 精度导致 near-singular 情况下调试结论不稳定。

Common mistakes include treating the type as authored USD data, assuming it stores translation, inverting without determinant checks, mixing row-major and column-major layouts, and ignoring `float` precision.

调试时可先打印四个元素、determinant、transpose 和 inverse，再和预期二维基向量逐项核对。

For debugging, compare the four elements, determinant, transpose, inverse, and expected 2D basis vectors directly.
