逐段双语理解 / Paragraph-Level Bilingual Coverage

官方简述说 `UsdImagingNurbsPatchAdapter` 是 “Delegate support for `UsdGeomNurbsPatch`”。这句话的重点不是新增一个建模 schema，而是说明本类位于 USD Imaging 的 adapter 层：它把已经由 `UsdGeomNurbsPatch` 描述的 USD prim 转换成 `UsdImagingDelegate`、Hydra render index 或 scene-index 数据源可以消费的数据。作者在 USD 文件中写入 NURBS patch 时仍面对 `UsdGeomNurbsPatch`；渲染、预览或成像同步时，`UsdImagingNurbsPatchAdapter` 才介入。

The official summary says this class provides delegate support for UsdGeomNurbsPatch. In this bilingual edition, that means the class belongs to the USD Imaging adapter layer and translates authored UsdGeomNurbsPatch prim data into data consumable by UsdImagingDelegate, Hydra render-index paths, or scene-index data sources.

继承关系上，本页应和 `UsdImagingPrimAdapter`、`UsdImagingGprimAdapter` 以及 `UsdImagingInstanceablePrimAdapter` 一起读。基类负责通用 prim 生命周期、路径、可见性、实例化和 gprim 共享行为；`UsdImagingNurbsPatchAdapter` 只补足 NURBS patch 特有的数据读取和失效语义。不要把它误读成 NURBS 曲面求值器、tessellation 算法库或最终 renderer 的网格生成实现。

The class should be read together with UsdImagingPrimAdapter, UsdImagingGprimAdapter, and UsdImagingInstanceablePrimAdapter. Those base adapters cover common prim lifecycle and gprim behavior, while this adapter covers the NURBS-patch-specific imaging contract.

`Data access` 分组中的 `GetPoints()` 和 `GetTopology()` 是本页最直接的读取入口。`GetPoints()` 面向控制点或点数据，`GetTopology()` 面向 patch 的拓扑描述；它们的职责是把 `UsdGeomNurbsPatch` 的几何输入整理给成像层，而不是直接返回某个 renderer 最终绘制的三角网格。排查 NURBS patch 不显示时，应同时确认 USD prim 上的 authored 数据是否存在、adapter 是否支持该 prim、Hydra 后端是否消费对应数据，以及 renderer 是否支持后续曲面细分或转换。

The Data access group exposes GetPoints() and GetTopology(). These methods provide patch geometry inputs to the imaging layer; they do not by themselves define the final renderer tessellation result.

`GetImagingSubprims()`、`GetImagingSubprimType()` 和 `GetImagingSubprimData()` 属于更现代的数据源式读取路径：一个 USD prim 可以在 imaging 层暴露为一个或多个 subprim，每个 subprim 有类型和数据。对 NURBS patch 来说，这帮助 adapter 把 patch 的几何、拓扑、可能的 primvar 或定位信息拆成 Hydra 可以按需查询的数据单元。阅读这些方法时，应保留 `TfToken`、`SdfPath`、`HdDataSourceLocatorSet` 等标识原样，因为它们是跨 USD 和 Hydra 边界传递语义的键。

GetImagingSubprims(), GetImagingSubprimType(), and GetImagingSubprimData() describe the data-source style subprim interface. Keep TfToken, SdfPath, and HdDataSourceLocatorSet identifiers unchanged because they are the semantic keys crossing the USD/Hydra boundary.

`Populate()` 是把某个 `UsdPrim` 引入 imaging 索引或 delegate 管理结构的入口；`IsSupported()` 则是支持性判断。若一个 `UsdGeomNurbsPatch` 在 stage 中存在但没有出现在成像结果里，先不要直接假设是几何数据损坏：应检查 `IsSupported()`、prim path、purpose/visibility 过滤、delegate population 顺序、实例化上下文以及后端 renderer 对 patch 数据的支持状态。

Populate() introduces a UsdPrim into the imaging index/delegate flow, while IsSupported() answers whether this adapter can handle a candidate prim. Missing output can come from support checks, path filtering, population order, instancing context, or renderer capabilities.

`TrackVariability()` 和 `ProcessPropertyChange()` 对应时间变化和属性变更传播。前者记录点、拓扑或相关属性是否随 `UsdTimeCode` 变化；后者把 USD 属性变化翻译成 Hydra dirty state、locator invalidation 或重新同步范围。若时间轴播放时 patch 形状没有更新，调试重点通常不是重新读取整个 stage，而是检查 variability 是否被正确标记、property change 是否到达 adapter，以及相应 subprim 或 locator 是否被失效。

TrackVariability() and ProcessPropertyChange() connect time-varying USD data to Hydra change propagation. They determine whether changing points/topology-related data causes the correct dirtying or invalidation work.

`InvalidateImagingSubprim()` 面向 subprim/data-source 缓存失效。它和传统 dirty bit 的目的相似，都是避免成像层继续使用过期数据，但表达粒度更贴近数据源 locator。对本页来说，关键是区分三件事：USD authored data 是否变化、adapter 是否把变化映射到正确的数据单元、Hydra 或 renderer 是否重新拉取并使用这些数据。

InvalidateImagingSubprim() invalidates cached subprim/data-source information. The practical distinction is between authored USD changes, adapter-side mapping, and Hydra/renderer re-consumption of the changed data.

NURBS patch 的阅读重点还包括它和普通 polygon mesh 的差异：patch 通常依赖控制点、阶数、结点向量、参数范围以及可能的 trim 或 primvar 信息来表达曲面，而不是直接列出三角面索引。`UsdImagingNurbsPatchAdapter` 的价值在于把这些高阶曲面语义稳定地带入 imaging 管线，并尽量保持 `UsdGeomNurbsPatch` 的 authored intent。若下游只看到空对象、错误边界框或静态旧形状，排查顺序应从 authored schema 数据、adapter data access、variability/dirtying、subprim locator 失效、Hydra 后端支持和最终 renderer 曲面处理能力逐层缩小。

A NURBS patch differs from a polygon mesh: the authored surface semantics are expressed through higher-order patch data rather than direct triangle lists, and this adapter preserves those semantics as data moves into the imaging pipeline.

在大型场景中，本 adapter 也影响性能判断。NURBS patch 可能比 mesh 更依赖按需取数和缓存失效，盲目把所有变化都当作完整 resync 会放大开销；但若 locator 或 dirty state 太窄，又会导致 Hydra 继续使用旧控制点或旧拓扑。阅读 `TrackVariability()`、`ProcessPropertyChange()` 和 `InvalidateImagingSubprim()` 时，要把“正确性”和“最小化更新范围”同时放在脑中：前者保证画面不陈旧，后者避免每帧重新构建不必要的数据。

For large scenes, the adapter also affects performance tradeoffs: change tracking must be broad enough for correctness but narrow enough to avoid unnecessary full resync work.

和相邻类型协作时，`UsdImagingDelegate` 负责把 stage、时间码和索引同步串起来，`UsdImagingNurbsPatchAdapter` 负责回答“这个 NURBS patch prim 该贡献哪些成像数据”。链接中出现的 `HdMeshTopology`、`HdBasisCurvesTopology`、`UsdGeomPrimvar` 等类型可作为对照：它们提示 Hydra 数据模型能承载多种几何拓扑和 primvar 表达，但不能反推出本页就是 mesh 或 basis curves 的 adapter。正确读法是保留 NURBS patch 的 schema 身份，再看它如何被投递到通用成像框架中。

Related topology and primvar types are useful comparisons, but they should not change the identity of this page: it remains the imaging adapter for UsdGeomNurbsPatch.

  • `GetPoints()`、`GetTopology()` 是 NURBS patch 数据读取重点，应同 `UsdGeomNurbsPatch` 的 schema 属性一起核对。GetPoints() and GetTopology() are the NURBS patch data-reading focus.

边界与常见误读 / Boundaries and Common Misreadings

  • 不是 authoring API：创建或编辑 NURBS patch prim 时看 `UsdGeomNurbsPatch`，不是本 adapter。This is not the authoring API; use UsdGeomNurbsPatch for authored schema data.
  • 不是 renderer：本类提供 imaging 数据桥接，最终曲面求值、细分、网格化或显示质量仍取决于 Hydra 后端和 renderer。This is not the renderer; final evaluation and display depend on the Hydra backend and renderer.
  • 不是孤立类：它要和 `UsdImagingDelegate`、`UsdPrim`、`SdfPath`、`TfToken`、`HdDataSourceLocatorSet` 以及相关 `UsdImaging*Adapter` 类型一起理解。Read it with UsdImagingDelegate, UsdPrim, SdfPath, TfToken, HdDataSourceLocatorSet, and related UsdImaging adapter classes.
  • 不是通用 mesh adapter：虽然最终显示可能经过网格化，输入语义仍是 NURBS patch，不应直接套用 `UsdGeomMesh` 的 topology 假设。Do not read it as a generic mesh adapter; the authored input semantics remain those of a NURBS patch.
  • 排障时应先确认 authored patch 数据，再检查 adapter 同步和 Hydra 消费；不要只看最终画面来判断 USD 数据是否错误。Debug authored patch data, adapter sync, and Hydra consumption before judging the USD data from the final image alone.
  • 适合清晰定位曲面成像链路问题。Useful for locating surface-imaging pipeline issues.
