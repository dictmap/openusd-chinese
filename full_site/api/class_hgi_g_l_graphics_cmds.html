逐段双语理解 / Paragraph-Level Bilingual Coverage

`HgiGLGraphicsCmds` 是 `HgiGraphicsCmds` 的 OpenGL backend implementation。官方一句 “OpenGL implementation of HgiGraphicsCmds” 的重点不是说明它是一种 USD scene object，而是说明它把 Hgi 的跨后端图形命令抽象落到具体 OpenGL 状态和 draw submission 上。它属于 HgiGL backend 层，位于 Hydra task 和 GPU driver 之间；它不 author USD layer，不创建 prim，也不解释 scene composition。

OpenGL implementation of HgiGraphicsCmds.

阅读这个类时，最重要的边界是 `HgiGraphicsCmds` 与 `HgiGLGraphicsCmds` 的关系：前者定义跨后端 graphics command encoder 的接口，后者负责把这些命令翻译为 GL pipeline binding、resource binding、vertex buffer binding、viewport / scissor state、draw call、memory barrier 和 debug marker。调用方通常通过 Hgi 抽象层取得 command object，而不是把它当成通用业务对象长期保存。

The class is the OpenGL-specific command encoder behind the Hgi graphics command abstraction.

`BindPipeline()` 是 draw 前最关键的状态入口之一。它绑定 `HgiGraphicsPipelineHandle` 指向的图形管线，使后续 `Draw()` / `DrawIndexed()` / indirect draw 使用正确的 shader program、raster state、blend/depth state 和 vertex layout。排查黑屏或状态串扰时，应先检查 pipeline handle 是否有效、是否属于同一个 HgiGL device，以及它的 vertex input layout 是否与后续 `BindVertexBuffers()` 匹配。

BindPipeline() establishes the graphics pipeline state used by later draw commands.

`BindResources()` 与 `BindVertexBuffers()` 分别处理 shader resource bindings 和 vertex buffer bindings。前者对应 textures、buffers、samplers、uniform/descriptor-like resource sets 在 GL 后端中的绑定；后者对应顶点输入流。它们描述的是 GPU 后端状态，不是 USD 属性连接，也不是 Hydra scene index 中的 data source locator。若材质、纹理或顶点数据异常，调试路径应从 resource set、buffer offset、stride、binding index 和 GL object lifetime 入手。

BindResources() and BindVertexBuffers() install resource bindings and vertex input buffers for subsequent draws.

`SetViewport()` 和 `SetScissor()` 控制 framebuffer 空间里的 rasterization region。它们与 USD camera aperture、`UsdGeomCamera`、Hydra framing 或 viewport UI 不是同一层概念：camera 决定投影和视图矩阵，HgiGL graphics commands 决定当前 GL draw 在 render target 中写入哪个矩形区域。常见错误是 framebuffer 尺寸变化后没有同步 viewport，或者 scissor 仍裁剪在旧区域。

SetViewport() and SetScissor() define the rasterization region in framebuffer coordinates.

`Draw()`、`DrawIndexed()`、`DrawIndirect()` 与 `DrawIndexedIndirect()` 是本页的执行核心。普通 draw 直接由参数指定 vertex/index count；indexed draw 依赖 index buffer；indirect draw 则依赖 GPU buffer 中的 command layout。indirect 版本出错时，不能只看 draw call 本身，还要检查 indirect command buffer 的数据布局、offset、buffer visibility 和前置 compute / copy pass 是否已经完成。

Draw(), DrawIndexed(), DrawIndirect(), and DrawIndexedIndirect() submit different forms of graphics draw work.

`SetConstantValues()` 处理短小的 per-draw constant data。它通常用于向当前 pipeline 传递小块常量，而不是替代完整 resource binding 或 USD attribute authoring。若常量不生效，要同时检查 pipeline layout、shader 中的 expected binding、offset/size、以及提交前是否已绑定正确的 pipeline。

SetConstantValues() provides small constant data for the active pipeline state.

`InsertMemoryBarrier()` 是 HgiGL 后端暴露 GPU memory visibility 语义的入口。它并不是 CPU mutex，也不是 USD 层面的 thread-safety 机制；它用于同一帧或相邻 GPU pass 中先写后读、image/buffer 状态转换、或 compute / graphics 之间可见性同步。看到上一 pass 的旧 buffer 或旧 texture 内容时，应检查是否缺少合适的 barrier。

InsertMemoryBarrier() inserts OpenGL memory-visibility synchronization for GPU resources.

`PushDebugGroup()`、`PopDebugGroup()` 和 `InsertDebugMarker()` 面向 frame capture、GPU debugger 和 backend diagnostics。它们不会改变 USD scene，也不会修复 draw state；它们提供的是可读的调试分组和标记。良好的调试记录通常会把 pipeline binding、resource binding、draw call、barrier 和 resolve/submit 边界清楚分组。

Debug groups and markers annotate command streams for graphics debuggers and frame captures.

`_Submit()` 是 protected member，表示 HgiGL 后端把记录好的 commands 提交给 OpenGL 执行的边界。`HgiGL` 被声明为 friend，说明生命周期和提交时机由 HgiGL 后端协调，而不是由上层应用随意调用。调试 command object 复用问题时，要区分 command recording、backend submission、GL object lifetime 和 Hydra task scheduling。

_Submit() is the OpenGL backend submission boundary, coordinated by HgiGL.

与相邻类型的关系也要分清：`HgiGraphicsCmds` 是跨后端接口，`HgiGLGraphicsCmds` 是 GL 实现；`HgiCmds` 是更基础的 command base；`HgiGLDevice` 管理 GL device/context 相关资源；`HgiGraphicsCmdsDesc` 描述创建命令对象时的 render target / descriptor 需求；`GfVec4f` 与 `GfVec4i` 常用于 viewport、clear color 或 scissor rectangle 等数值状态。它们都是渲染后端概念，不是 USD composition arcs。

Related types describe the command abstraction, GL device resources, command descriptors, and numeric state values.

常见误读包括：把 `HgiGLGraphicsCmds` 当作 Hydra render task；把 `Draw()` 失败归因于 USD prim authoring；把 viewport/scissor 当成 camera 设置；把 memory barrier 当成 CPU lock；把 debug marker 当成执行控制。正确的排查顺序通常是：确认 HgiGL context 与 device 有效，确认 pipeline/resource/vertex binding 匹配，确认 viewport/scissor 覆盖目标 framebuffer，确认 draw/indirect 参数和 buffer layout 正确，确认必要的 memory barrier 与 submit 顺序存在。

Common debugging separates backend command state from USD authoring, scene indexing, camera setup, and Hydra scheduling.
