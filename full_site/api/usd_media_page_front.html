逐段双语理解 / Paragraph-Level Bilingual Coverage

UsdMedia 是 OpenUSD 中用于在 stage 上下文里包含其他媒体的 schema 模块。官方 Overview 的第一句话很短：它提供一种 representation，用来在 stage context 中包含例如 audio 这样的媒体。这里的关键不是“播放媒体”，而是“把媒体作为场景描述的一部分放进 USD 结构中”，让资产预览、音频位置、非空间音频以及场景层级可以被同一套 composition、asset resolution 和 prim 组织方式管理。UsdMedia represents other media, such as audio, in the context of a stage.

官方当前只列出两类媒体类型。第一类是 UsdMediaAssetPreviewsAPI，它把预渲染的 asset previews 编码到 scene 中的 prim 上，例如 thumbnails。中文读者应把它理解成资产浏览、内容库、DCC 预览和发布管线的辅助信息：预览图帮助人和工具快速识别资产，但它不是最终渲染输出，也不是 UsdRender 的 render product。UsdMediaAssetPreviewsAPI encodes pre-rendered asset previews, such as thumbnails, on scene prims.

第二类是 UsdMediaSpatialAudio，它把 spatial 和 non-spatial audio 编码为 scene 中派生自 UsdGeomXformable 的 prim。这个继承关系非常重要：如果是 spatial audio，位置、层级、transform 和场景空间关系会影响读者如何理解它；如果是 non-spatial audio，它仍然可以作为 scene prim 被组织和发布，但不一定代表声源来自某个空间位置。UsdMediaSpatialAudio encodes spatial and non-spatial audio as UsdGeomXformable-derived prims.

本页不应被误读成音频引擎、视频播放系统、codec 封装、实时媒体流协议或渲染器输出管线。UsdMedia 提供的是 schema-level scene description：媒体是什么、挂在哪个 prim 或场景上下文中、是否有空间意义、预览资源如何与资产相关联。实际解码、播放、混音、流式传输、UI 展示和浏览器缩略图缓存，属于下游工具或运行时。UsdMedia is schema-level scene description, not a playback engine, codec layer, or renderer output system.

因此，阅读 UsdMedia 的主路径是：先判断媒体是 asset preview 还是 audio；再判断它是绑定到资产 prim 的预览信息，还是作为可变换 prim 出现在场景中的音频对象；然后检查资源路径、stage composition、prim 层级、transform、asset resolver 和下游消费工具。只有这样，读者才能把“USD 描述正确但播放器不支持”“资源路径失效”“preview 不是最终渲染图”“spatial audio 位置不对”等问题分层定位。The practical reading path separates asset previews, spatial audio, stage composition, asset resolution, transforms, and consumer support.

schema 分组与职责 / Schema Groups and Responsibilities

  • UsdMediaAssetPreviewsAPI：用于把预渲染的 asset previews 关联到 scene prim。典型用途是 thumbnails、内容浏览卡片、资产库搜索结果、发布审核和 DCC 工具里的快速预览。它回答“这个资产有什么代表性预览图或预览媒体”，不回答“renderer 应该如何生成最终图像”。UsdMediaAssetPreviewsAPI attaches pre-rendered asset previews to scene prims.
  • UsdMediaSpatialAudio：用于把 spatial 和 non-spatial audio 表达成 UsdGeomXformable 派生 prim。它让音频能参与 USD 场景组织，尤其是需要空间位置、层级或 transform 的音频对象。UsdMediaSpatialAudio represents audio as UsdGeomXformable-derived prims.
  • stage context：官方强调媒体包含在 stage context 中，说明媒体不是孤立文件列表，而是和 prim、layer、reference、payload、variant、asset resolution 一起进入场景描述。Stage context means media is organized with the scene, not just listed as external files.
  • thumbnail 与 asset preview：缩略图或预览媒体是发布和浏览辅助材料，通常由外部工具生成；USD 负责描述它们与资产或 prim 的关系，下游工具负责读取和展示。Thumbnails and previews are auxiliary published media read by downstream tools.

从数据责任看，UsdMedia 位于“场景描述”和“内容工具”之间。它既不像 UsdGeom 那样定义几何拓扑，也不像 UsdRender 那样定义渲染输出产品，还不像播放器那样负责音频采样、混音或解码。它提供的价值，是让媒体资源能以 USD schema 的方式被组合、定位、发布和审计。UsdMedia sits between scene description and content tooling, rather than geometry, render output, or playback runtime.

对资产库来说，AssetPreviewsAPI 可以让同一个资产 prim 带上缩略图、turntable preview 或其他预渲染媒体，便于用户在不打开完整场景、不执行渲染的情况下识别内容。对虚拟制作或交互场景来说，SpatialAudio 则让音频线索和空间布局放在一起，便于工具在读取 stage 时找到音频 prim 与几何、相机、动画或 UI 之间的关系。Asset previews help browsing; spatial audio keeps audio cues inside the scene organization.

边界、误读点与调试路径 / Boundaries, Misreads, and Debugging

  • 不要把 UsdMedia 当作完整媒体播放框架。它不会保证某个应用能播放音频、显示缩略图或解码视频；这些能力取决于工具、插件、运行时和平台 codec。UsdMedia is not a complete media playback framework.
  • 不要把 AssetPreviewsAPI 里的 thumbnail 当作 UsdRender 的最终 render product。thumbnail 是资产预览或管理信息，最终图像输出仍应看 UsdRender、Hydra 或 renderer 相关设置。Thumbnails are not final render products.
  • 不要把所有 audio 都理解为空间声源。官方明确说 SpatialAudio 可编码 spatial 和 non-spatial audio；是否有空间语义，要结合 prim 位置、transform 和具体 schema 属性继续看。UsdMediaSpatialAudio can encode both spatial and non-spatial audio.
  • 不要把 UsdGeomXformable 派生关系理解成音频一定可见。它说明音频 prim 可以进入场景空间和层级系统，不表示它会被几何渲染器绘制成可见对象。UsdGeomXformable placement does not make audio a visible geometric object.
  • 不要把 UsdUI 的界面提示和 UsdMedia 的媒体资源混在一起。UI hint 可以帮助工具展示属性或 prim，媒体 schema 则描述预览资源和音频对象。UsdUI hints and UsdMedia media schemas are separate concerns.

推荐调试顺序是：先确认目标 prim 是否在最终 composed stage 中存在；再确认媒体 schema 或 API 是否被正确应用；随后检查预览资源或音频资源的 asset path、resolver、权限和相对路径；若是 spatial audio，再检查 UsdGeomXformable transform、父子层级和坐标空间；最后检查下游工具是否支持读取对应 schema、是否能显示 preview 或播放 audio。Debug from composed prims through schema application, asset paths, transforms, and downstream tool support.

如果预览图不显示，问题可能在资源不存在、resolver 找不到、preview API 没有挂在预期 prim、工具只支持部分 preview 类型，或总入口/本地链接没有正确路由。若音频听不到，问题可能在资源路径、音频格式、播放工具支持度、spatial/non-spatial 语义、transform、timeline 或 runtime audio system。中文页必须把这些层分开，避免把所有问题都归咎于 OpenUSD schema。Preview and audio failures should be separated into USD authoring, asset resolution, schema support, and runtime playback.

发布流程中还要记录快照属性：预渲染 previews 可能反映某个时间点的资产外观，如果资产后续修改而 preview 没有再生成，用户看到的 thumbnail 可能已经过期。因此 UsdMediaAssetPreviewsAPI 的存在不代表预览一定是最新的，也不代表它和 renderer 当前输出完全一致。审核时应把 preview 当作辅助证据，而不是权威渲染结果。Pre-rendered previews are snapshots and may become stale when assets change.

管线发布与资源治理 / Pipeline Publishing and Asset Governance

在实际管线里，UsdMedia 页面最容易被低估，因为它不像几何、材质或渲染设置那样直接影响画面。但是对资产库和协作系统来说，预览图和音频资源经常是用户第一次接触资产的入口。如果 AssetPreviewsAPI 挂错 prim、预览资源路径失效、缩略图和资产版本不一致，用户会在打开完整 stage 之前就拿到错误信号。因此媒体 schema 的质量检查应和资产发布检查一起做，而不是等到渲染或播放阶段才发现。UsdMedia metadata is often the first asset-facing signal in content libraries and review tools.

对 AssetPreviewsAPI，建议发布时记录预览资源的生成来源、对应资产版本、分辨率或用途层级，并确保 asset resolver 能从本地站、离线包或生产环境中解析这些资源。预览图可以很小，但语义不能含糊：它是展示模型外观、材质状态、动画姿态、变体选择，还是只是占位缩略图。中文用户如果只看到 thumbnail 一词，很容易误以为它是权威渲染结果，所以本页明确把 preview 定位为辅助浏览和审核材料。Asset previews should be versioned and resolved as auxiliary browsing and review material.

对 SpatialAudio，发布检查应同时看 USD 结构和运行时消费。USD 侧要确认 audio prim 的层级、transform、命名和引用关系；资源侧要确认音频文件可解析、格式被目标工具支持；运行时侧要确认播放器或引擎是否真正读取 spatial/non-spatial 语义。一个音频 prim 能在 stage 中被发现，并不等于每个查看器都会播放它，也不等于空间衰减、循环、同步或混音策略已经统一。SpatialAudio validation must separate USD structure, media resource availability, and runtime playback support.

这类边界说明对本地中文站很重要：如果只保留官方短句，读者会知道有两个类型，却不知道该怎么排错。完成页必须把“描述层”和“消费层”分开：USD 描述媒体和关系，asset resolver 找资源，DCC 或浏览器展示预览，音频系统播放或混音，渲染系统只在需要时引用相关信息。把这些层分开后，错误归因才不会在 UsdMedia、UsdRender、UsdGeom 和下游工具之间来回漂移。The Chinese path separates description, asset resolution, preview display, audio playback, and rendering concerns.

因此，本轮晋级不仅翻译 Overview，还把读者从 API 模块入口带到 release user guide 的 AssetPreviewsAPI、SpatialAudio、usdMedia overview 和目录页；同时保留到 Usd、UsdGeom、UsdRender、UsdUI 的相邻路径。这样用户在本地中文站中可以顺着阅读路径判断：当前问题是媒体 schema 表达、资源解析、空间变换、预览展示、音频播放，还是 UI/渲染工具消费能力。The promoted page links module scope to release schema pages and adjacent API modules.

版本治理上，媒体资源还要和资产快照一起审计。预览文件可能来自旧版本，音频文件可能被移动，stage 中的 prim 可能通过 reference 或 variant 切换到另一套资产。检查 UsdMedia 时应记录资源路径、生成时间、所属变体、目标工具和已验证的消费端；否则本地中文站即使能打开页面，也无法帮助用户判断媒体信息是否仍可信。Media resources should be audited with asset versions, variants, paths, and verified consumers.

用户实际阅读时，可以从总入口进入 API，再由本页跳到 release 的 usdMedia schema 页面；若预览或音频仍有问题，再返回本页按资源、空间、工具消费三层排查。这样本页承担的是连续阅读和问题分流入口，而不是孤立的 Doxygen 摘要，并且能支持后续人工复审与发布回归检查，减少媒体资源误配导致的排查成本、重复沟通、返工与风险增加。The local page acts as a reading and triage entry, not just a Doxygen summary.
