逐段双语理解 / Paragraph-Level Bilingual Coverage

本页已按完整双语参考页整理：中文优先解释 `PcpErrorUnresolvedPrimPath` 作为 Pcp composition / asset resolution 错误对象的职责、resolved 与 loaded 两个阶段的区别、`site`、`unresolvedPath`、`sourceLayer`、`targetLayer`、`arcType` 等字段如何定位失败来源，以及 `New()` 与 `ToString()` 在结构化诊断和日志输出中的边界；英文类名、字段名、方法名、类型名、链接和官方摘录继续保留，便于与 Doxygen 原页逐项核对。

This page is promoted to a complete bilingual reference. It keeps official API names, data-member names, method names, type names, links, and excerpts while adding Chinese-first coverage of this Pcp unresolved-prim-path error.

官方摘录说，这个错误表示 asset paths could not be both resolved and loaded。这里的 resolved 是路径或资产标识能被 resolver 解释到某个候选资源；loaded 是该资源还能被 Pcp / Sdf 实际打开并用于 composition。两者必须同时满足，任一阶段失败都可能触发这类 unresolved prim path 诊断。

Asset paths that could not be both resolved and loaded.

`PcpErrorUnresolvedPrimPath` 继承自 `PcpErrorBase`，它不是普通 C++ exception，也不是最终 `UsdStage` 上的属性值。它是 Pcp 在计算 prim index 或沿 composition arc 追踪资产时生成的结构化错误对象，用于告诉调用方某个 prim path 或引用目标无法在当前 resolver context、layer stack 和 composition site 下完成解析与加载。

`PcpErrorUnresolvedPrimPath` is a structured Pcp composition error derived from `PcpErrorBase`.

`site` 字段指向发生错误的 `PcpSite`。它把错误固定到 composition graph 中的某个站点，而不仅仅是最终 stage path。排查时应从 `site` 反查所在 layer stack、prim path 和相关 node；只看最终 `UsdPrim` 路径，通常无法解释错误是从哪个 reference、payload、inherit 或 specialize 站点传入的。

The `site` member identifies the composition site where the unresolved-prim-path error was detected.

`unresolvedPath` 是未能解析或加载的 `SdfPath`。这个字段应与 asset resolver 日志、layer identifier、authored reference 或 payload target 一起阅读。它不是文件系统路径字符串；它表示 scene description 中的 prim path 目标，因此路径拼写、namespace remapping 和 `PcpMapExpression` 都可能影响它最终是否可达。

The `unresolvedPath` member records the `SdfPath` for the prim path that could not be resolved and loaded.

`sourceLayer` 与 `targetLayer` 用来描述错误跨 layer 发生的位置。`sourceLayer` 通常帮助定位 authored opinion 来自哪里，`targetLayer` 则帮助确认目标资源或被加载 layer。源层存在该 opinion，并不保证目标层能被 resolver 找到或能被 Sdf 成功打开；这也是 resolved 和 loaded 必须分开检查的原因。

The `sourceLayer` and `targetLayer` members help identify the layer relationship involved in the failed resolution or load.

`arcType` 记录触发该错误的 `PcpArcType`。如果 arc 是 reference，通常检查 referenced asset path 和 target prim；如果是 payload，需要同时考虑 payload loading policy；如果是 inherit 或 specialize，则更应检查目标 prim path 与 layer stack 中的 authored relationship。不同 arc type 对应不同排查入口。

The `arcType` member identifies the composition arc type associated with the unresolved prim path.

`New()` 是创建该错误对象的静态入口，通常由 Pcp 内部在检测到对应错误时使用。调用者消费的是返回的错误对象及其字段，而不是通过 `New()` 去修复路径。`~PcpErrorUnresolvedPrimPath()` 只是析构生命周期的一部分，不表示错误已被处理或目标路径已恢复。

`New()` constructs the error object; object construction is separate from repairing the unresolved asset or prim path.

`ToString()` 提供人类可读的错误描述，适合日志、报告和 UI 展示；但自动化审计和调试工具不应只保存字符串。保留 `site`、`unresolvedPath`、`sourceLayer`、`targetLayer` 与 `arcType` 这些结构化字段，才能把错误重新关联到 prim index、layer stack、resolver context 和具体 composition arc。

`ToString()` provides human-readable diagnostics, while the structured fields remain important for tooling.

常见误用包括：把 unresolvedPath 当成文件路径；只检查资产是否存在而不检查 target prim path 是否存在；把 resolved 成功误认为 loaded 必然成功；忽略 `arcType` 导致用 reference 的方式排查 payload；或者只看 `ToString()` 而丢失 `sourceLayer` / `targetLayer`。正确做法是沿 `site` 和 `arcType` 回到 authored opinion，再结合 resolver 日志和 layer 可加载性逐层排查。

Common misuse includes treating the unresolved prim path as a filesystem path, ignoring the arc type, or collapsing resolved and loaded into one check.
