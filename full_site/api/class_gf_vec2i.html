逐段双语理解 / Paragraph-Level Bilingual Coverage

`GfVec2i` 是 `Gf` Linear Algebra 模块中的二维 `int` 向量值类型。官方描述里的 “Basic type for a vector of 2 int components” 说明它只保存两个 `int` 分量，通常用于整数坐标、像素尺寸、网格索引、二维离散偏移或计数型数据；它不是 USD scene prim、不是 attribute 容器，也不携带 stage / layer / schema 语义。

`GfVec2i` is a basic value type for a vector of two `int` components. It is not a USD scene object or attribute container.

“fast and simple” 是理解该类的设计边界。它是轻量数学值类型，成员通常直接围绕分量读写、简单构造、比较、算术运算和线性代数辅助函数展开；读者不应期待它进行复杂单位系统转换、自动归一化、浮点插值或几何拓扑管理。需要连续几何精度时，应选择或转换到 `GfVec2f`、`GfVec2d` 等浮点向量类型。

The phrase “fast and simple” means this class is a lightweight value type. Use floating-point vector types when continuous precision or interpolation is needed.

`ScalarType` 对应 `int`，这是本页最重要的 typedef。它决定了 `operator+`、`operator-`、`operator*`、`operator+=`、`operator*=` 等 arithmetic operators 的结果仍按整型分量理解。若把浮点缩放、单位向量、投影结果或长度计算强行放回 `GfVec2i`，就要意识到整数截断、舍入或转换规则可能改变数学意义。

`ScalarType` is `int`, so arithmetic and storage semantics are integer-based. Conversions from floating-point results must be handled deliberately.

构造函数 `GfVec2i() [1/4]` 到 `[4/4]` 表示该类支持多种初始化路径：默认构造、用两个整数分量构造、从数组或其他兼容向量类型构造。阅读这些 overload 时应保留 `GfVec2i()`、`ScalarType` 和参数类型原名，因为 C++ overload resolution、隐式转换和精度变化是调试数值 bug 时的关键线索。

The constructor overloads cover default construction, component-wise construction, and compatible array/vector inputs. Overload and conversion details matter for numeric debugging.

`data()`、`GetArray()` 和 `operator[]` 暴露底层两个分量的访问方式。它们适合与要求连续小数组的底层数学或图形代码交互，但不表示这个对象是 USD attribute array。`operator[]` 的索引通常只对 0 和 1 有意义；越界访问、把分量顺序误当成 row/column matrix 语义，或把它与 `SdfPath` / attribute path 混淆，都是常见误读。

`data()`, `GetArray()`, and `operator[]` expose the two stored components. They are component accessors, not USD attribute-array APIs.

`GetLengthSq()` 返回平方长度，常用于避免开方的比较，例如判断哪个整数偏移更短、网格半径是否超过阈值、或在排序中比较距离。对 `GfVec2i` 而言，平方长度通常比 normalized length 更自然，因为两个 `int` 分量不能精确表达多数单位向量；如果需要 `sqrt` 后的连续长度，应在调用侧转换到浮点类型或使用合适的 `GfVec2f` / `GfVec2d`。

`GetLengthSq()` is useful for squared-distance comparisons and avoids treating integer vectors as normalized floating-point vectors.

`GfDot()`、`GetProjection()` 和 `GetComplement()` 说明 `GfVec2i` 仍处在向量代数体系中，但投影和补向量的数学含义要结合整型结果理解。点积本身对整数向量很直接；投影和补向量若涉及除法或浮点中间量，则调用方应检查返回类型、截断边界和是否需要改用浮点向量。

Dot products are straightforward for integer vectors, while projection and complement operations require attention to return type and conversion boundaries.

`Axis()`、`XAxis()`、`YAxis()` 和静态 public attributes 可用于构造标准基向量，例如 `(1,0)` 或 `(0,1)`。这些值常用于二维数组方向、tile/grid 遍历、图像坐标偏移或工具算法中的轻量常量；它们不是 transform node，不携带空间单位，也不代表 USD 坐标系中的 authored orientation。

Axis helpers provide basis vectors for indexing, grid traversal, and integer-coordinate algorithms, not authored scene transforms.

比较运算如 `operator==` / `operator!=` 按两个分量逐项比较。由于 `GfVec2i` 是整型值类型，这种比较是精确的；这与 `GfVec2f` / `GfVec2d` 中常见的 epsilon 或浮点误差讨论不同。若数据来自浮点计算后再转换为 `int`，真正需要检查的是转换策略，而不是比较运算本身。

Equality for integer vectors is exact component-wise comparison. Conversion from floating-point values is usually the part that needs scrutiny.

常见调试路径是：先确认两个分量的含义是 `(x,y)`、`(width,height)` 还是 generic index pair；再确认 `ScalarType=int` 是否适合后续运算；随后检查 `operator[]` 的索引、数组传递的分量顺序、`GetLengthSq()` 的溢出风险，以及是否误把 `GfVec2i` 当成具有连续精度的几何向量使用。API 名称、`int`、`GfVec2f`、`GfVec2d`、`GfDot()` 等英文标识应原样保留。

A practical debugging path is to verify component meaning, integer semantics, indexing order, squared-length overflow risk, and whether a floating-point vector type is more appropriate.
