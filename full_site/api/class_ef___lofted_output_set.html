逐段双语理解 / Paragraph-Level Bilingual Coverage

`Ef_LoftedOutputSet` 是 Ef 执行系统中的内部集合类型，用来代表 `EfPageCacheBasedExecutor` 跟踪一组 lofted outputs。官方描述里的 lofted output 指 evaluation 期间其值来自 page cache 的 output；这里的 output 是 `VdfOutput` 语义，不是 USD layer 中的 authored property，也不是 Hydra render output。阅读本页时应把它放在 Ef/Vdf 执行缓存、依赖传播和失效清理的上下文中，而不是放在场景描述 authoring API 中。

`Ef_LoftedOutputSet` tracks the set of lofted outputs, meaning outputs whose values were sourced from the page cache during evaluation, on behalf of `EfPageCacheBasedExecutor`.

“sourced from the page cache” 是本页最关键的边界：当前 evaluation 使用了缓存页中已有的 `VdfOutput` 值，因此执行器需要记住这些输出，以便后续依赖收集、缓存刷新或节点失效时知道哪些结果来自缓存。这个集合描述的是 executor runtime state；它不会改变 `UsdStage`、`SdfLayer`、schema 属性或任何 authored scene description。

The page-cache wording makes the class part of executor runtime bookkeeping, not authored USD scene data.

`Add()` 与 `Remove()` 维护单个 output 的成员关系。调用方不仅要关心 `VdfOutput` 本身，还要关心对应的 `VdfMask`，因为同一个输出在 Vdf evaluation 中可能只对某个 mask 范围有效。把 output 当成无 mask 的全量值，会误读该集合在依赖传播中的精度。

`Add()` and `Remove()` maintain membership for outputs, with mask information carried by `VdfMask`.

`CollectLoftedDependencies()` 是本页最值得关注的方法：它把已经从 page cache 取得的 lofted outputs 转换为当前 evaluation 需要考虑的依赖信息。换句话说，即使值来自缓存，执行系统仍然需要知道这些缓存值依赖哪些输入、节点和网络状态；否则重新求值、增量更新或失效判断会丢失必要的因果关系。

`CollectLoftedDependencies()` gathers dependency information for the lofted outputs so cached values still participate in dependency analysis.

`RemoveAllOutputsForNode()` 按 `VdfNode` 清理集合中的相关输出，适用于节点失效、网络重建、executor cache refresh 或删除节点输出时的维护场景。它说明 `Ef_LoftedOutputSet` 与 `VdfNode` / `VdfId` / `VdfNetwork` 的关系是运行时索引关系，而不是持久的资源所有权关系。

`RemoveAllOutputsForNode()` removes outputs associated with a node, supporting invalidation and executor maintenance.

`Resize()`、`Clear()` 和 `GetSize()` 展示了这个类的集合生命周期。`Resize()` 允许集合容量或索引空间随网络/执行状态调整，`Clear()` 清空本轮记录，`GetSize()` 暴露当前集合规模；这些方法都指向一个可重建、可清理的 executor 辅助结构，而不是稳定存档格式。

`Resize()`, `Clear()`, and `GetSize()` describe collection lifecycle, capacity, reset, and size queries for runtime bookkeeping.

本页关联的 `VdfNetwork`、`VdfOutput`、`VdfMask`、`VdfId` 和 `VdfNode` 应保留英文原名。`VdfNetwork` 提供被执行的计算网络，`VdfNode` 是网络节点，`VdfOutput` 是节点上的输出端，`VdfMask` 限定输出值范围，`VdfId` 支持内部标识和索引。`Ef_LoftedOutputSet` 的职责是在这些类型之间记录“哪些输出被从缓存抬升进当前求值”的集合关系。

Related identifiers such as `VdfNetwork`, `VdfOutput`, `VdfMask`, `VdfId`, and `VdfNode` should remain unchanged because they name the exact API types.

常见误读包括：把 lofted output 翻译成 USD 文件中的“输出属性”；把 page cache sourced value 理解成新生成的 authored value；忽略 `VdfMask` 导致依赖范围过宽；把 `RemoveAllOutputsForNode()` 当成删除场景节点；以及把本页当成普通用户 authoring API。正确理解是：这是 Ef executor 内部用于 page-cache based evaluation 的 dependency/provenance bookkeeping。

Common misreadings are to treat lofted outputs as scene-layer outputs, authored data, or rendering outputs. The class is instead Ef executor bookkeeping for page-cache based evaluation.
