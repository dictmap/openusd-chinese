逐段双语理解 / Paragraph-Level Bilingual Coverage

本页已按完整双语参考页整理：中文优先解释 `PcpArc` 在 Pcp prim index graph 中表示 composition arc 的职责、source node 与 target parent node 的方向、`PcpArcType`、`PcpNodeRef`、`PcpMapExpression` 等字段的边界，以及它与 authored reference/payload/inherit/specialize/variant opinion、USD namespace 层级和运行期 `UsdPrim` 访问之间的区别；英文类名、字段名、类型名、链接和官方摘录继续保留，便于与 Doxygen 原页逐项核对。

This page is promoted to a complete bilingual reference. It keeps official API names, data-member names, type names, links, and excerpts while adding Chinese-first coverage of Pcp composition-arc semantics.

官方定义说，`PcpArc` 表示 prim index 中连接两个 nodes 的 arc。该 arc 由一个 node 拥有，这个拥有者是 source，并指向 index 中它的 parent node，作为 target。这里的 source/target 方向是 Pcp composition graph 内部的边方向，不是 USD stage 中 prim 父子关系的几何边，也不是 layer 文件中单独 authored 的 reference 或 payload 语句。

`PcpArc` represents an arc connecting two nodes in the prim index. The arc is owned by a node, the source, and points to its parent node, the target, in the index.

`type` 保存 `PcpArcType`，用于描述这条 composition arc 的来源类别，例如 reference、payload、inherit、specialize 或 variant 等组合关系。调试 prim index 时应先看 `type`，再看 source node、parent node 和路径映射；只知道某个 prim 有 opinion，不足以判断它在组合强弱顺序中的位置。

The `type` data member stores the `PcpArcType` that identifies the kind of composition relationship represented by the arc.

`parent` 是目标 parent node 的 `PcpNodeRef`。它告诉你 source node 通过这条 arc 连接到 index 中的哪一个父节点。这个 parent 是 composition graph 的父节点，不应直接等同于 `SdfPath` 的 textual parent，也不应等同于 `UsdPrim::GetParent()` 返回的运行期 namespace 父 prim。

The `parent` member identifies the parent node, or target, in the prim index graph.

`origin` 记录这条 arc 的 origin 信息，用于解释当前边由哪个组合站点或更早的 arc 链路产生。分析复杂 inherits、references 或 variants 时，`origin` 能帮助追踪“这条边为什么出现”；它不是 scene description 中的 authored object，也不是用于编辑 USD 文件的 handle。

The `origin` member records origin information that helps trace how this composition arc was introduced.

`mapToParent` 是 `PcpMapExpression`，描述 source node namespace 如何映射到 parent node namespace。它是理解 reference、payload 或 variant 进入当前 prim index 后路径如何重定位的关键字段。路径映射错误会影响 prim 和 property opinions 的落点，因此应与 `PcpPropertyIndex`、`SdfPath` 和 layer stack 一起诊断。

The `mapToParent` member is a `PcpMapExpression` describing how namespace paths map from the source node to the parent node.

`namespaceDepth` 与命名空间映射相关，但它不是 prim path 字符串长度，也不是 scene graph 的可视层级深度。它服务于 Pcp 内部组合计算，用来保持 namespace 映射和遍历判断的上下文；调试时应把它看作 composition bookkeeping，而不是用户可直接编辑的 USD 数据。

The `namespaceDepth` member belongs to Pcp's internal composition bookkeeping for namespace mapping.

`siblingNumAtOrigin` 表示在 origin 处与同源 sibling arcs 相关的顺序信息。组合图中多条 arc 可能来自同一个 origin 或相邻的 composition site；这个字段帮助保持稳定顺序和诊断 sibling 关系。它不是 child index，也不是 USD namespace 中第几个 child prim 的通用编号。

The `siblingNumAtOrigin` member records sibling-ordering information at the arc origin.

`PcpArc()` 构造的是一个低层组合图数据结构实例。创建或交换 `PcpArc` 不会向 layer 写入 reference、payload 或 inherits；真正的 authored opinions 仍存在于 `SdfLayer` / `SdfPrimSpec` / `SdfPropertySpec` 等 scene description 数据中。`PcpArc` 只是 Pcp cache 计算 prim index 后用来表达组合边的结果结构。

Constructing a `PcpArc` creates a composition-graph data structure, not authored scene description.

阅读本页时应和 `PcpPrimIndex`、`PcpNodeRef`、`PcpPropertyIndex` 一起看：prim index 汇总图，node 表示具体 composition site，arc 描述 node 之间的组合边，property index 则在属性级别列出 opinion sites。把这些层次分开，才能解释某个 property 或 prim opinion 为什么来自某个 layer、为什么被 stronger opinion 覆盖，或为什么路径映射后落到另一个 namespace。

Read `PcpArc` together with `PcpPrimIndex`, `PcpNodeRef`, and `PcpPropertyIndex` to understand the full composition structure.

常见误用包括：把 `PcpArc` 当成 USD layer 里的 authored arc；把 `parent` 当成 `UsdPrim` 的直接父对象；把 `mapToParent` 当成字符串拼接规则；把 `namespaceDepth` 当成路径长度；或忽略 `type` 只看 source/target。正确做法是先确认 `type`，再结合 `parent`、`origin`、`mapToParent` 和 sibling ordering 分析 prim index 的组合路径。

Common misuse includes treating `PcpArc` as authored USD data, as a namespace edge, or as a plain string-mapping record.
